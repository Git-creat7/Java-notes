# 回顾
## Malloc
`malloc` 的全称是 **Memory Allocation**（内存分配）。它是 C 语言中从**堆（Heap）**区域手动申请内存的核心函数
```C
	void* malloc(size_t size);
```
- **参数 `size`**：想要申请的内存字节数。通常配合 `sizeof` 使用，以确保跨平台兼容性。
    
- **返回值**：
    
    - 成功：返回分配到的内存**首地址**（`void*` 类型，可以强转为任何类型）。
    - 失败：返回 `NULL`

| **特性**  | **栈 (Stack)** | **堆 (Heap) / malloc** |
| ------- | ------------- | --------------------- |
| **大小**  | 较小（通常几 MB）    | 很大（取决于系统内存）           |
| **控制权** | 系统自动分配/释放     | 程序员手动分配/释放            |
| **灵活性** | 固定大小，编译时确定    | **动态确定**，运行中可变        |

# 线性表
由 n(n>0)个数据特性相同的元素构成的有限序列，称为线性表。
- 线性表是 n 个数据元素的有限序列，其中 n个数据是相同数据类型的
- 线性表中元素的个数 n(n≥0)定义为线性表的长度，当 n=0 时称之为空表。
- 对于非空的线性表或线性结构，其特点是:
	- **存在唯一的一个被称作“第一个”的数据元素**
	- **存在唯一的一个被称作“最后一个”的数据元素**
	- **除第一个元素外，结构中的每个数据元素均只有一个前驱**
	- **除最后一个元素外，结构中的每个数据元素均只有一个后继**

## 顺序表
用一组连续的内存单元依次存储线性表的各个元素，也就是说，逻辑上相邻的元素，实际的物理存储空间也是连续的。

## 链表
- 链表中的结点是独立的对象，在内存中是不连续的，每个节点包含数据值和下一个结点的地址
- 链表==查询慢==，无论查询哪个数据都从头开始找。
- 链表的==增删快==（对比数组）
### 节点
在链表中，数据不再是连续存放的，而是分散在内存的各个角落。每个数据块被称为一个**节点**。
一个节点由两部分组成：
1. **数据域**：存想存的东西（如 `int`）。
    
2. **指针域**：存**下一个节点**的地址（那个“钩锁”）
```C
Node* initList(){
    Node* head = (Node*)malloc(sizeof(Node));
    head -> data = 0;
    head -> next = NULL;
    return head;
}
```
别名：增加可读性
```C
typedef struct Node {
    ElemType data;
    struct Node *next;
} Node, *LinkedList; // LinkedList 等价于 Node*
```
如果这样写了，`main` 函数里就可以写成： `LinkedList list = initList();` 
这样从字面上看，`list` 就代表了“整个链表”，而不仅仅是一个节点指针了。
**但在底层，它依然是一个 `Node*`**

### 头插法
为了保证链表不断裂，这两行代码的**顺序**是绝对不能错的：

1. `p -> next = L -> next;`
    
    **逻辑**：让新节点 `p` 先牵住原先排在第一位的那个人。如果后面原本没节点（`NULL`），那么 `p->next` 就指向 `NULL`。
    
2. `L -> next = p;`
    
    **逻辑**：让头结点 `L` 改手牵住新节点 `p`
```C
int insertHead(Node* L,ElemType e){
	Node* p = (Node*)malloc(sizeof(Node));
	p -> data = e;
	p -> next = L -> next;
	L -> next = p;
}
```
假设依次插入 `10`, `20`, `30`：
1. 插入 10：`Head -> 10 -> NULL`
    
2. 插入 20：`Head -> 20 -> 10 -> NULL`
    
3. 插入 30：`Head -> 30 -> 20 -> 10 -> NULL`
#### 无头结点的头插法 
```C
// 无头结点头插法，需要修改头指针，所以用二级指针
int insertHead(Node** L, ElemType e)
{
    Node* p = (Node*)malloc(sizeof(Node));
    if(p == NULL) return 0;
    p->data = e;

    p->next = *L;   // 新结点接原链表开头
    *L = p;         // 头指针更新为新结点
    return 1;
}
```
**无二级指针**
```C
// 传入旧头指针，返回插入后的新头指针
Node* insertHead(Node* L, ElemType e)
{
    // 新建结点
    Node* p = (Node*)malloc(sizeof(Node));
    p->data = e;

    // 头插核心
    p->next = L;   // 新结点指向原第一个结点
    L = p;         // 头指针更新为新结点

    return L;      // 返回新头结点
}
```

### 尾插法
要实现尾插，最关键的是找到当前链表的**最后一个节点**。

1. 创建一个指针 `r`（取名为 rear，意为尾部），初始指向头结点。
    
2. 通过 `while` 循环让 `r` 走到尽头（即 `r->next == NULL` 的地方）。
    
3. 把新节点挂在 `r` 的后面
```C
int insertTail(LinkList L,ElemType e){
    //创建新节点
    LinkList h = (LinkList)malloc(sizeof(Node));
    if(h == NULL) return 0;
    h -> data = e;
    h - > next = NULL;
    //找到尾节点
    Node* r = L;
    while(r -> next != NULL){
        r = r -> next;
    }
    //将新节点插入到尾节点后面
    r -> next = h;
    return 0;
}
```
假设链表现在是 `Head -> 10 -> NULL`，要插入 `20`：

- **找尾巴**：`r` 发现 `r->next` 指向 10，于是 `r` 变成 10。10 的 `next` 是 `NULL`，循环停止，`r` 现在就是 10。
    
- **挂载**：执行 `r->next = p`，即 10 的 `next` 指向了 20。
    
- **结果**：`Head -> 10 -> 20 -> NULL`

### 在指定位置插入
假设要将新元素 `e` 插入到第 `i` 个位置（从 1 开始计数）：
1. 找到第 `i-1` 个节点（即**目标位置的前驱节点**）。
    
2. 创建一个新节点 `s`。
    
3. 将 `s` 的 `next` 指向原先的第 `i` 个节点。
    
4. 将第 `i-1` 个节点的 `next` 指向 `s`
```C
int insertPos(LinkList L,ElemType e,int pos){
	 Node* m = L;//当前扫描到的节点
	 int j = 0 // 当前p指向的是第几个节点
	
	 while(m != NULL && j < pos - 1){
	     m = m->next;
	     j++;
	 }
	 if(m == NULL || pos < 1) return 0;
	 Node* p = (Node*)malloc(sizeof(Node));
	 if(p == NULL) return 0;
	 p -> data = e;
	 p -> next = m -> next;
	 m -> next = p;
	 return 1;
}
```
### 删除节点
假设要删除第 `i` 个节点：

1. 找到第 `i-1` 个节点（**前驱节点**），记为 `p`。
    
2. 定义一个临时指针 `q` 作为要删除的节点（即 `p->next`）。
    
3. 修改指针：让 `p->next` 指向 `q->next`（跳过被删除节点）。
    
4. 释放 `q` 节点的内存
```C
int deletePos(LinkList L,int pos,ElemType *e){
    Node* p = L;//当前扫描到的节点
    int j = 0; // 当前p指向的是第几个节点
    while(p -> next != NULL && j < pos - 1){
        p = p -> next;
        j++;
    }
    if(p -> next == NULL || pos < 1) return 0;

    Node* q = p -> next;//q作为被删除的节点
    *e = q -> data;//将被删除节点的数据保存到e中
    p -> next = q -> next;//将q的下一个节点链接到p上
    free(q);
    return 1;
}
```
### 释放链表
假设链表结构为：`L (头结点) -> A -> B -> C -> NULL`

- **`Node* p = L -> next;`**
    
    - **作用**：让指针 `p` 指向链表的第一个真正存放数据的节点（首元节点 `A`）。
        
    - **注意**：这里跳过了头结点 `L`，因为头结点通常不存数据，只需清空它后面的内容。
        
- **`Node* q;`**
    
    - **作用**：声明一个辅助指针 `q`。它是实现逻辑的关键，用来“探路”。
        
- **`while(p != NULL){ ... }`**
    
    - **作用**：只要还没走到链表末尾，就继续循环执行释放操作。
        
- **核心循环体内部：**
    
    1. **`q = p -> next;`**：在杀掉 `p` 之前，先让 `q` 指向 `p` 的下一个节点（比如 `B`）。**如果不这样做，一旦释放了 `p`，就再也找不到 `B` 了。**
        
    2. **`free(p);`**：释放当前节点 `p` 所占用的内存。
        
    3. **`p = q;`**：让 `p` 追上 `q` 的位置，准备处理下一个节点。
        
- **`L -> next = NULL;`**
    
    - **作用**：当所有数据节点（A, B, C）都释放完后，头结点 `L` 依然存在，但它现在指向的是已经被销毁的内存。
        
    - **目的**：这一步将头结点的指针域置为空。此时，链表在逻辑上变回了一个**空的带头结点链表**

```c
void freeList(LinkList L){
   if(L == NULL) return;
   Node* p = L -> next;
   Node* q;
   while(p != NULL){
       q = p -> next;
       free(p);
       p = q;
   }
   L -> next = NULL;
}
```

```C
#include <bits/stdc++.h>

typedef char ElemType;
typedef struct Node {
    ElemType data;
    struct Node *next;
} Node, *LinkList;

// 尾插法：创建一个带头结点的链表
LinkList createList_Tail(const char *s) {
    LinkList L = (LinkList)malloc(sizeof(Node));
    L->next = NULL;
    Node *r = L;
    for (int i = 0; s[i] != '\0'; i++) {
        Node *p = (Node *)malloc(sizeof(Node));
        p->data = s[i];
        r->next = p;
        r = p;
    }
    r->next = NULL;
    return L;
}

Node* findIntersectionNode(LinkList headA, LinkList headB) {
    // 安全检查
    if (headA == NULL || headB == NULL || headA->next == NULL || headB->next == NULL) return NULL;

    int lenA = 0, lenB = 0;
    // 从第一个数据节点开始计算长度
    Node* pA = headA->next;
    Node* pB = headB->next;

    while (pA != NULL) { lenA++; pA = pA->next; }
    while (pB != NULL) { lenB++; pB = pB->next; }

    int step = abs(lenA - lenB);
    
    // 重新指向首元节点
    pA = headA->next;
    pB = headB->next;

    // 让长链表先走
    if (lenA > lenB) {
        while (step--) pA = pA->next;
    } else {
        while (step--) pB = pB->next;
    }
    // 同步推进寻址
    while (pA != pB) {
        pA = pA->next;
        pB = pB->next;
    }
    return pA; // 如果不相交，最后 pA, pB 都会是 NULL，返回 NULL 也是正确的
}

int main(void) {
    // 1. 创建公共后缀
    LinkList common = createList_Tail("ing");
    
    // 2. 创建各自的前缀
    LinkList being = createList_Tail("be");
    LinkList loading = createList_Tail("load");

    // 3. 构造相交结构：将前缀末尾指向公共部分的第一个数据节点
    Node *p = being;
    while (p->next != NULL) p = p->next;
    p->next = common->next; 

    Node *q = loading;
    while (q->next != NULL) q = q->next;
    q->next = common->next;

    // 4. 调用函数
    Node* result = findIntersectionNode(being, loading);
    
    if (result != NULL) {
        printf("相交节点数据为: %c\n", result->data); // 输出 i
    } else {
        printf("未发现相交节点\n");
    }

    return 0;
}
```

### 链表的局限
| **局限性类别** | **具体表现描述 **                    | **常见的改进/替代方案 **               |
| --------- | ------------------------------ | ----------------------------- |
| **访问速度**  | 无法随机访问，查找第 $n$ 个节点需要 $O(n)$ 时间 | **数组**（随机访问）或 **跳表**（通过索引层加速） |
| **遍历方向**  | 只能向后走，无法直接获取“前驱”节点             | **双链表**（增加 `prev` 指针）         |
| **插入/删除** | 在已知节点前操作或删除当前节点时，仍需从头遍历找前驱     | **双链表** 或 **使用虚拟头节点**（简化边界操作） |
| **空间开销**  | 每个数据都需要额外存储一个指针空间              | **数组** 或 **静态链表**（在连续空间内模拟链表） |
| **可靠性**   | 链表一旦中间断裂（指针指向错误），后续数据全部丢失      | **双链表**（具备一定的冗余恢复能力）          |

## 循环链表
- 循环链表（Circular Linked List）是单链表的一个奇妙变体。
- 它最显著的特征是：**最后一个节点的指针不再指向 `NULL`，而是重新指向头节点**，从而形成一个闭合的环
- 在单链表中，我们判断遍历结束的条件是 `curr->next == NULL`；而在循环链表中，由于没有“尽头”，我们通常判断 `curr->next == head`

|**优势**|**描述**|**实际应用场景**|
|---|---|---|
|**首尾直达**|如果持有尾指针，可以在 $O(1)$ 时间找到头和尾。|队列的循环实现、缓冲区管理。|
|**无死角遍历**|从链表中任何一个节点出发，都可以访问到其他所有节点。|播放器的循环播放列表、多轮巡检任务。|
|**简化合并**|合并两个循环链表只需修改两个指针，无需遍历。|资源池的动态合并。|
### 找出环的起始节点
```C
LinkList isCycleLink(LinkList head){
    Node* fast = head;
    Node* slow = head;
    while (fast!=NULL && fast->next!=NULL){
            fast = fast->next->next;
            slow = slow->next;
            
            if(fast==slow){
	            //第三节点从头出发
                Node* y = head;
                while(y!=slow){
                    y=y->next;
                    slow=slow->next;
                }
                return y; // 返回入环节点
            }
                
    }
    return NULL;
}
```

## 双向链表
- 双向链表是链表的“完全体”。
- 它通过增加一个空间开销，彻底解决了单链表“无法回头”的痛点。
### 优势
- **真正的 $O(1)$ 删除**：
    
    在单链表中，如果要删除节点 `p`，必须从头遍历找到 `p` 的前驱。但在双向链表中，`p->prev` 直接就告诉了前驱是谁。
    
- **双向遍历**：
    
    你可以从头走到尾，也可以从尾走到头。这在处理某些需要倒序查看数据的算法（如最近最少使用缓存 LRU）时是刚需。
    
- **插入更灵活**：
    
    你可以在某个节点的前面或后面直接插入，而不需要像单链表那样总是要在“前一个节点”上做文章。
### 头插法

```C
LinkList insertHead(LinkList head,ElemType data){
    Node* node = (Node*)malloc(sizeof(Node));
    if(node == NULL) return head;
    node->data = data;
    node->next = head->next;
    if(head->next != NULL){
        head->next->prev = node;
    }
    node->prev = head;
    head->next = node;
    return head;
}
```
### 尾插法
```C
LinkList insertTail(LinkList head,Node* oldTail,ElemType data){
    Node* tail = (Node*)malloc(sizeof(Node));
    if(tail==NULL) return NULL;
    tail->data = data;
    tail->prev = oldTail;
    oldTail->next = tail;
    tail->next = NULL;
    return head;
}
```

```C
LinkList insertPos (LinkList head,int pos,ElemType data){
    Node* p = head;
    int j = 0;
    //寻找第pos-1的节点
    while(p != NULL && j <pos-1){
        p = p->next;
        j++;

    //超过链表长度
    if(p==NULL) return NULL;
    Node* node = (Node*)malloc(sizeof(Node));
    if(node==NULL) return NULL;
    node->data = data;

    // p     ->     pos 
    //      node
    node->next = p->next;
    p->next = node;
    p->next->prev = node;
    node->prev = p;
    return head;
}
```

### 删除节点
```C
// 双向链表：删除第 pos 个节点
// 参数：head 链表头结点，pos 删除位置，data 带回被删节点的值
// 返回：删除成功返回头结点，失败返回 NULL
LinkList deletePos(LinkList head, int pos, ElemType* data) {
    // 位置非法（小于1），直接返回
    if (pos < 1)
        return NULL;

    // p 指向第一个数据节点（带头结点）
    Node* p = head->next;
    int j = 1;  // j 记录当前节点的位置

    // 找到第 pos 个节点：p 向后走，直到 j == pos
    while (p != NULL && j < pos) {
        p = p->next;
        j++;
    }

    // 循环结束后 p == NULL，说明 pos 超出链表长度，删除失败
    if (p == NULL)
        return NULL;
    // 把被删除节点的数据保存到 *data 中带回
    *data = p->data;

    // 核心：双向链表删除节点
    // 1. 让前驱节点的 next 直接指向后继节点
    p->prev->next = p->next;

    // 2. 如果不是最后一个节点，让后继节点的 prev 指向前驱节点
    if (p->next != NULL) {
        p->next->prev = p->prev;
    }
    // 释放被删除节点的空间
    free(p);

    // 删除成功，返回链表头结点
    return head;
}

```

## 中缀转后缀

- 当左括号在栈外时，属于最高优先级，当左括号在栈里时，属于最低优先级
- 如果是右括号，且栈顶元素不是左括号，持续出栈并输出，直到顶为左括号出栈结束，最后将左括号也出栈
- 如果是运算符，判断优先级，如果优先级大于栈顶元素，压入栈中。否则将栈顶元素出栈输出，然后将当前运算符压入栈中

❗**左括号直接入栈，数字直接输出，运算符栈内比优先级，右括号弹出到左括号**
```C++
// 运算符优先级
int priority(char c){
    if(c == '+' || c == '-') return 1;
    if(c == '*' || c == '/') return 2;
    return 0; // 括号
}

// 中缀转后缀
void InfixToPostfix(string s){
    stack<char> st;
    for(char ch : s){
        // 1. 数字：直接输出
        if(isdigit(ch)){
            cout << ch;
        }
        // 2. 左括号：入栈
        else if(ch == '('){
            st.push(ch);
        }
        // 3. 右括号：弹出到左括号为止
        else if(ch == ')'){
            while(st.top() != '('){
                cout << st.top();
                st.pop();
            }
            st.pop(); // 扔掉左括号
        }
        // 4. 运算符
        else{
            // 栈顶优先级 >= 当前，先弹出
            while(!st.empty() && priority(st.top()) >= priority(ch)){
                cout << st.top();
                st.pop();
            }
            st.push(ch);
        }
    }
    // 剩余全部弹出
    while(!st.empty(){
        cout << st.top();
        st.pop();
    }
}

int main(){
    // 举例子：中缀 8/2+5-6
    string infix = "8/2+5-6";
    cout << "后缀表达式：";
    InfixToPostfix(infix);
    return 0;
}
```

## 树的定义
树是由 $n (n \ge 0)$ 个节点组成的有限集合。

- **根节点 (Root)**：树顶部的节点，没有父节点。
    
- **子节点 (Child) 与 父节点 (Parent)**：直接相连的上下级关系。
    
- **叶子节点 (Leaf)**：没有子节点的节点（度为 0）。
    
- **深度 (Depth)**：从根到该节点的路径长度。
    
- **高度 (Height)**：从该节点到最远叶子节点的路径长度


| **概念**            | **解释**                            |
| ----------------- | --------------------------------- |
| **节点的度 (Degree)** | 该节点拥有的子树个数。                       |
| **树的度**           | 树中所有节点度的最大值。                      |
| **层数 (Level)**    | 根为第 1 层，其子节点为第 2 层，以此类推。          |
| **深度 (Depth)**    | 从根节点开始**自顶向下**累加（根到该节点的路径长度）。     |
| **高度 (Height)**   | 从叶子节点开始**自底向上**累加（该节点到最远叶子的路径长度）。 |
| **森林 (Forest)**   | $m$（$m \ge 0$）棵互不相交的树的集合。         |
|                   |                                   |
- **节点与边的关系**：在一棵拥有 $n$ 个节点的树中，总边数恰好为 $n-1$。
    
    > _原理：除了根节点外，每个节点都有且仅有一条指向它的边。_
    
- **度与节点的关系**：树中所有节点的度之和等于 $n-1$[root]（即总边数）

- 对于度为 m 的树，**第 i 层上最多有 $m^{i-1}$ 个结点。

- 对于高度为 h，度为 m 的树，最多 $(m^{h}-1)/m-1$ 有个结点。

## 二叉树
**二叉树（Binary Tree）** 是 $n$（$n \ge 0$）个结点的有限集合。
它或者是空集（称为空二叉树），或者由一个**根结点**及两棵互不相交的、分别称为左子树和右子树的二叉树组成。

**核心要点：**

- **度限制**：每个结点最多只能有 **2** 棵子树（即度 $\le 2$）。
    
- **有序性**：左子树和右子树是有顺序的，次序不能任意颠倒。即使某个结点只有一棵子树，也要区分它是左子树还是右子树

**性质一：**
- 在二叉树的第 $i$ 层上，最多有 $2^{i-1}$ 个节点（规定根节点为第 1 层）
**性质二：**
- 深度为 $k$ 的二叉树，最多有 $2^k - 1(k>=1)$ 个节点。
**性质三：**
- 对于任何一棵二叉树，如果其叶子节点（度为 0）数为 $n_0$，度为 2 的节点数为 $n_2$，则有：

$$n_0 = n_2 + 1$$

> **重要性**：**无论树的形状如何，只要是二叉树就成立。**
### 满二叉树
- **定义**：
	- 在一棵二叉树中，所有分支结点都存在左子树和右子树，并且所有叶子都在同一层上。
		
	- 深度为 k 且含有 $2^k - 1$ 个节点的二叉树
	
- **特点**：
    
    - 结构极其对称。
        
    - 相同深度的二叉树中，满二叉树的节点个数最多（为 $2^k - 1$）。
        
    - 不存在度为 1 的节点
	
### 完全二叉树

- **定义**：对一棵具有 $n$ 个结点的二叉树按层序编号，如果编号为 $i$（$1 \le i \le n$）的结点与同样深度的满二叉树中编号为 $i$ 的结点在二叉树中位置完全相同，则称为完全二叉树。
    
- **特点**：
    
    - **叶子只能出现在最下两层**，且底层的叶子均依次集中在左侧。
        
    - 它是满二叉树的“前身”，满二叉树一定是完全二叉树，反之不然。
        
    - **意义**：最适合用**数组**存储，不会浪费空间
	
- **没有左子树，不能有右子树，上一层没有铺满，不能有下一层**
	
-  **满二叉树只从最后一层最右侧逐个删节点，得到的一定是完全二叉树**
	
- **性质：**
	- 具有 n 个结点的完全二叉树的深度为 $(\lfloor \log_2 n \rfloor + 1)$（向下取整）

```C
// 创建二叉树（前序遍历规则）
// 参数：TreeNode **T  二级指针
// 作用：修改 指针本身 的指向（必须用二级指针才能在函数内修改外面的指针）
void createBiTree(TreeNode **T)
{
    ElemType ch;   // 定义存储输入字符的变量
    cin >> ch;     // 输入节点值（# 代表空节点）

    // 如果输入 #，表示这个位置没有节点，置为空
    if(ch == '#')
        *T = NULL;      // *T 代表 真正的节点指针，让它指向空

    // 否则：创建节点 + 赋值 + 递归创建左右子树
    else{
        // 1. 申请一个树节点大小的内存
        // *T 指向这个新节点（真正修改了外部指针的指向）
        *T = (BiTree)malloc(sizeof(TreeNode));

        // 2. 给新节点赋值
        (*T)->data = ch;

        // 3. 递归创建 左子树
        // &(*T)->left  等价于：取左孩子指针的地址
        // 传进去依然是二级指针，和函数参数匹配
        createBiTree(& (*T)->left);

        // 4. 递归创建 右子树
        createBiTree(& (*T)->right);
    }
}
```
### 前序遍历

- **顺序：根节点 -> 左子树 -> 右子树**
    
- **描述：** 只要遇到节点，先访问它，然后递归地去处理它的左子树，最后处理右子树。
    
- **特点：** 根节点总是第一个被访问的。

### 中序遍历 

- **顺序：左子树 -> 根节点 -> 右子树**
    
- **修正：** 并不是简单地“从最左边开始”，而是**先完整地遍历左子树**，再访问当前节点，最后遍历右子树。
    
- **特点：** 对于**二叉搜索树 (BST)**，中序遍历的结果是一个严格递增的序列。

### 后序遍历

- **顺序：左子树 -> 右子树 -> 根节点**
    
- **修正：** 它的逻辑是**先完成左右子树的访问**，最后才回到当前节点。
    
- **特点：** 根节点总是最后一个被访问的。常用于文件目录的大小计算或删除整棵树。
    

###  层序遍历 

- **顺序：按层从上到下，每层从左到右**
    
- **描述：** 借助**队列 (Queue)** 实现。先将根节点入队，然后依次弹出节点，并将该节点的左右子节点入队。

|**遍历方式**|**访问顺序简记**|**核心逻辑**|
|---|---|---|
|**前序**|**根** - 左 - 右|遇到就拿，先根后后|
|**中序**|左 - **根** - 右|左边到底，中间插一杠|
|**后序**|左 - 右 - **根**|左右搞定，最后回根|
|**层序**|按层遍历|广度优先 (BFS)|
### 二叉树遍历性质
- 已知前序遍历和中序遍历，可以唯一确定一棵二叉树
- 已知中序遍历和后序遍历，可以唯一确定一棵二叉树
- **已知前序遍历和后序遍历，是不能确定一棵二叉树的**

## 线索二叉树

在普通的二叉树链表表示法中，会有大量的**空指针**（$n$ 个节点的树有 $n+1$ 个空链域）。线索化就是把这些浪费的空间利用起来，指向该节点在某种遍历序列中的**前驱**或**后继**

### 中序遍历
**线索化二叉树**
```C
struct ThreadNode {

   ElemType data;
   ThreadNode *lchild, *rchild;
   int ltag, rtag; // 0: 指向左右孩子, 1: 指向互补的前驱/后继
   
}ThreadNode, *ThreadTree;

ThreadNode *pre = nullptr;

void inOrderThread(ThreadTree T){
    if(T == nullptr) return;
    inOrderThread(T->lchild); // 递归线索化左子树
    //线索化当前节点
    if(T->lchild == nullptr){
        T->lchild = pre; //左边没孩子了，直接之前前驱
        pre->ltag = 1; //前驱线索
    }

    if(pre!=nullptr && pre->rchild == nullptr){
        pre->rchild = T; //前驱的右边没孩子了，直接当前节点
        pre->rtag = 1; //后继线索
    }

    pre = T; //更新前驱为当前节点
    inOrderThread(T->rchild); // 递归线索化右子树
}

// 封装一个启动函数，处理最后一个节点的收尾
void CreateInThread(ThreadNode *root) {
    pre = nullptr;
    if (root != nullptr) {
        InThreading(root);
        pre->rchild = nullptr; // 最后一个节点后继为空
        pre->rtag = 1;
    }
}
```

| --------- | -------------------- | --------------------------------------------------- |

| **寻找起点**  | 极其简单                 | `return T; // 根节点就是第一个`                             |
| **寻找后继**  | 优先找孩子                | `1.有左找左; 2.无左找右; 3.无孩走线索`                           |

- **缺**：找前驱（向前遍历）极难，除非有父指针，否则无法直接通过线索找前驱

## 哈夫曼树
- 哈夫曼树（Huffman Tree），又称**最优二叉树**，是数据结构中一种非常经典的贪心算法应用。
- 它的核心目标是：在已知一组叶子节点及其权重（权值）的情况下，构建一棵二叉树，使得这棵树的**带权路径长度（WPL）最小**
**WPL**
- **路径长度**：从根节点到该节点的路径上的分支数目。
    
- **结点的带权路径长度**：该结点的权值 $\times$ 它的路径长度。
    
- **树的带权路径长度 (WPL)**：树中所有**叶子节点**的带权路径长度之和
$$WPL = \sum_{i=1}^{n} w_i l_i$$
（其中 $w_i$ 是第 $i$ 个叶子的权值，$l_i$ 是其路径长度）

### 构造哈夫曼树
构造过程非常直观，可以概括为：**“每次选两个最小的，合体成一个新根。”**

**具体步骤：**

1. **初始化**：将 $n$ 个权值看作 $n$ 棵只有一个节点的孤立小树，组成一个森林。
    
2. **找最小**：在森林中挑选两棵**根节点权值最小**的树。
    
3. **合并**：
    
    - 创建一个新节点作为这两个节点的父节点。
        
    - 父节点的权值 = 左右孩子权值之和。
        
    - 从森林中删除那两个最小的，把新合成的树放回森林。
        
4. **重复**：重复步骤 2 和 3，直到森林中只剩下一棵树。这棵树就是哈夫曼树
### 哈夫曼编码
哈夫曼编码（Huffman Coding）是哈夫曼树最直接的应用，也是数据压缩（如 JPEG、ZIP）的核心原理之一。

它的核心逻辑是：**用最短的比特（Bit）去代表出现频率最高的字符。**

- 从**根节点**出发，向下走向目标叶子节点。
    
- 每向**左**走一步，记录一个 **`0`**。
    
- 每向**右**走一步，记录一个 **`1`**。
    
- 到达叶子节点时，所得的 `01` 序列就是该字符的哈夫曼编码

#### 特性A. 前缀性质（Prefix Property）

这是哈夫曼编码能成功解码的关键。**任何一个字符的编码都不是另一个字符编码的前缀。**

- _例子：_ 如果 `A` 是 `01`，那么绝对不会有字符的编码是 `011` 或 `010`。
    
- _好处：_ 在解码时，你从左往右读，只要匹配到一个序列，就可以立即确定是哪个字符，不会产生“歧义”。
    

#### 特性B. 最优性

在所有可能的变长编码中，哈夫曼编码能使传输电文的总长度达到**最小**

## 二叉查找树
- 每一个节点上最多有两个子节点
- 任意节点左子树上的值都小于当前节点
- 任意节点右子树上的值都大于当前节点

### 添加节点
- 小的存左边，大的存右边，一样的不存
### 弊端
- 左子树全部为空，从形式上看，更像一个==单链表==
- 插入速度没有影响
- 查询速度明显降低

### 解决弊端：平衡二叉树
平衡二叉树也叫 **平衡二叉搜索树（Self-balancing binary search tree）**，又被称为 AVL 树，可以保证 **查询效率较高**。它是解决 **二叉排序** 可能出现的查询问题。

 - 二叉查找树
 - 规则：==**任意**节点左右子树高度差不超过1==

- 右旋

- 步骤：
	- 以不平衡的点作为支点
	- 把支点左旋降级，变成左子节点
	- 晋升 原来的右子节点

- 一次左旋

- 先局部右旋 -> 右右RR
- 再整体左旋

## 红黑树
- 平衡二叉树的弊端：在添加节点的时候，旋转次数太多，造成添加节点的时间浪费
- 红黑树刚开始被称为**平衡二叉B树**
- 是一种特殊的二叉查找树，红黑树的每一个节点上都有存储位表示节点的颜色
- <span style="color: #dc3545;">每一个节点可以是红或者黑；红黑树不是高度平衡的，他的平衡是通过“红黑规则“进行实现的</span>
- 是一个二叉查找树
- ==但是不是高度平衡的==
- 条件：特有的<span style="color: #dc3545;">红黑规则</span>
### 红黑规则
1. 每一个节点是红色或黑色
2. 根节点必须是黑色的
3. 如果一个节点没有子节点或者父节点，则该节点相应的指针属性值为Nil，这些Nil被视为叶节点，每个叶节点 (Nil) 是黑色的
4. 如果某一个节点是红色的，那么它的子节点必须是黑色（不能出现两个红色节点相连的情况）
5. 对每一个节点，从该节点到其所有后代叶节点的简单路径上，均包含相同数目的黑色节点

#### 添加节点
- 添加的节点默认是红色的（红色效率高）
- 查找、插入、删除的时间复杂度最坏为O(log n)
