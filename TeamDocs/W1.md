## Java 参数校验（Bean Validation）

### 常用注解

| 注解                 | 说明                                  |
| ------------------ | ----------------------------------- |
| `@NotNull`         | 值不能为 `null`，但允许空字符串                 |
| `@NotEmpty`        | 值不能为 `null` 且长度 > 0（适用于字符串、集合）      |
| `@NotBlank`        | 值不能为 `null`，且去除空白字符后长度 > 0（仅适用于字符串） |
| `@Size(min, max)`  | 字符串长度 / 集合大小必须在 `[min, max]` 范围内    |
| `@Min(value)`      | 数值最小值                               |
| `@Max(value)`      | 数值最大值                               |
| `@Pattern(regexp)` | 字符串必须匹配正则表达式                        |
| `@Email`           | 字符串必须是合法 Email 格式                   |

### 示例

```java
@NotBlank(message = "登录账号不能为空")
@Size(min = 2, max = 16, message = "账号长度必须在2-16之间")
private String username;

@NotBlank(message = "密码不能为空")
private String password;
```

> 注意：同一字段重复标注相同注解（如两个 `@NotBlank`）是多余的，只保留一个即可。

### 三者区别

```
@NotNull  → null 不通过，"" 通过，" " 通过
@NotEmpty → null 不通过，"" 不通过，" " 通过
@NotBlank → null 不通过，"" 不通过，" " 不通过  ✅ 最严格，推荐用于字符串字段
```

### 使用步骤

1. **引入依赖**（Spring Boot 已内置）

2. **在 DTO/实体类字段上加注解**（如上示例）

3. **在 Controller 方法参数上加 `@Valid` 或 `@Validated`**

   ```java
   @PostMapping("/login")
   public Result login(@RequestBody @Valid LoginDTO dto) {
       // ...
   }
   ```

4. **全局异常处理**（捕获校验失败异常）

   ```java
   @RestControllerAdvice
   public class GlobalExceptionHandler {

       @ExceptionHandler(MethodArgumentNotValidException.class)
       public Result handleValidException(MethodArgumentNotValidException e) {
           String msg = e.getBindingResult().getFieldErrors().stream()
               .map(FieldError::getDefaultMessage)
               .collect(Collectors.joining("; "));
           return Result.fail(msg);
       }
   }
   ```

### `@Valid` vs `@Validated`

| | `@Valid` | `@Validated` |
|---|---|---|
| 来源 | JSR-303 标准 | Spring 扩展 |
| 分组校验 | 不支持 | 支持 |
| 嵌套校验 | 支持（加在字段上） | 支持 |
| 常用场景 | 一般校验 | 需要分组时使用 |

# Spring Security
**Spring Security 本质就是“拦截请求 + 校验身份 + 判断权限”**

Spring Security 是 Spring 生态中最核心的安全框架，用于为 Java / Spring Boot 应用提供：

- 用户认证（Authentication）
- 权限控制（Authorization）
- 防御常见攻击（CSRF、Session Fixation 等）
- JWT / OAuth2 / SSO / LDAP 集成

通常用于：

- 后端 REST API
- 管理系统
- 企业级系统
- 微服务认证中心
- 单点登录（SSO）

---

## 核心概念

| 概念                   | 说明                                     |
| -------------------- | -------------------------------------- |
| **认证**               | 验证"你是谁"，即登录验证                          |
| **授权**               | 验证"你能做什么"，即权限控制                        |
| `SecurityContext`    | 保存当前用户认证信息的容器（线程级别）                    |
| `Authentication`     | 表示已认证的用户信息（主体 + 凭证 + 权限）               |
| `UserDetails`        | 框架用于描述用户的接口，需自行实现                      |
| `UserDetailsService` | 根据用户名加载用户信息的接口，核心扩展点                   |
| `PasswordEncoder`    | 密码加密/校验接口，推荐使用 `BCryptPasswordEncoder` |

---

## 认证流程（表单登录 / JWT）

```Java
请求 → UsernamePasswordAuthenticationFilter
         ↓
     AuthenticationManager（ProviderManager）
         ↓
     AuthenticationProvider（DaoAuthenticationProvider）
         ↓
     UserDetailsService.loadUserByUsername()
         ↓
     PasswordEncoder.matches()
         ↓
     认证成功 → 存入 SecurityContextHolder
```

## 1. Authentication（认证）

“你是谁？”

验证用户身份。

例如：

- 用户名密码登录
- JWT Token
- OAuth2 登录
- 微信/Google/GitHub 登录

例如：

```Java
	@Bean
	public UserDetailsService userDetailsService() {
	    UserDetails user = User.withDefaultPasswordEncoder()
	            .username("admin")
	            .password("123456")
	            .roles("ADMIN")
	            .build();
	
	    return new InMemoryUserDetailsManager(user);
	}
```

## 2. Authorization（授权）

“你能访问什么？”

比如：

- 普通用户只能访问 `/user`
- 管理员才能访问 `/admin`

示例：

```Java
	http.authorizeHttpRequests(auth -> auth
	        .requestMatchers("/admin/**").hasRole("ADMIN")
	        .requestMatchers("/user/**").hasRole("USER")
	        .anyRequest().authenticated()
	);
```

## 工作原理
核心思想：

> 基于 Filter（过滤器链）

```
	请求
	 ↓
	Security Filter Chain
	 ↓
	认证
	 ↓
	权限检查
	 ↓
	Controller
```

**核心部件**

| 组件                      | 作用     |
| ----------------------- | ------ |
| `SecurityFilterChain`   | 安全过滤器链 |
| `Authentication`        | 当前登录用户 |
| `UserDetailsService`    | 查询用户   |
| `PasswordEncoder`       | 密码加密   |
| `SecurityContextHolder` | 保存登录信息 |
```Java
	@Bean
	public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
	
	    http
	        .csrf(csrf -> csrf.disable())
	        .authorizeHttpRequests(auth -> auth
	            .anyRequest().authenticated()
	        )
	        .formLogin(Customizer.withDefaults());
	
	    return http.build();
	}
```

## 3. UserDetails & UserDetailsService

Spring Security 通过 `UserDetailsService` 加载用户，需要自行实现。

### 实现 UserDetails（封装用户信息）

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class LoginUser implements UserDetails {

    private User user;

    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        // 将权限字符串转为 GrantedAuthority 集合
        return user.getPermissions().stream()
                .map(SimpleGrantedAuthority::new)
                .collect(Collectors.toList());
    }

    @Override public String getPassword()               { return user.getPassword(); }
    @Override public String getUsername()               { return user.getUsername(); }
    @Override public boolean isAccountNonExpired()      { return true; }
    @Override public boolean isAccountNonLocked()       { return true; }
    @Override public boolean isCredentialsNonExpired()  { return true; }
    @Override public boolean isEnabled()                { return true; }
}
```

### 实现 UserDetailsService（查询数据库）

```java
@Service
public class UserDetailsServiceImpl implements UserDetailsService {

    @Autowired
    private UserMapper userMapper;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        User user = userMapper.selectByUsername(username);
        if (user == null) {
            throw new UsernameNotFoundException("用户名或密码错误");
        }
        return new LoginUser(user);
    }
}
```

## 4. JWT 集成（前后端分离）

前后端分离项目一般关闭 Session，使用 JWT 做无状态认证。

### 整体流程

```
登录请求
  ↓
手动调用 AuthenticationManager.authenticate()
  ↓
认证通过 → 生成 JWT → 返回给前端
  ↓
后续请求携带 JWT（Header: Authorization: Bearer xxx）
  ↓
JwtAuthenticationFilter 解析 Token → 写入 SecurityContextHolder
  ↓
Controller 正常处理
```

### 登录接口

```java
@RestController
public class LoginController {

    @Autowired
    private AuthenticationManager authenticationManager;

    @PostMapping("/login")
    public Result login(@RequestBody @Valid LoginDTO dto) {
        UsernamePasswordAuthenticationToken authToken =
                new UsernamePasswordAuthenticationToken(dto.getUsername(), dto.getPassword());

        // 触发认证，内部会调用 UserDetailsService + PasswordEncoder
        Authentication authentication = authenticationManager.authenticate(authToken);

        LoginUser loginUser = (LoginUser) authentication.getPrincipal();
        String jwt = JwtUtil.createJWT(loginUser.getUser().getId().toString());
        return Result.ok(jwt);
    }
}
```

### JWT 认证过滤器

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private RedisCache redisCache;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        String token = request.getHeader("Authorization");
        if (!StringUtils.hasText(token)) {
            // 没有 token，放行（后续由权限拦截）
            filterChain.doFilter(request, response);
            return;
        }

        // 解析 token → 获取 userId → 从 Redis 取用户信息
        String userId = JwtUtil.parseJWT(token).getSubject();
        LoginUser loginUser = redisCache.getCacheObject("login:" + userId);

        // 写入 SecurityContextHolder
        UsernamePasswordAuthenticationToken authToken =
                new UsernamePasswordAuthenticationToken(loginUser, null, loginUser.getAuthorities());
        SecurityContextHolder.getContext().setAuthentication(authToken);

        filterChain.doFilter(request, response);
    }
}
```

### SecurityFilterChain 配置（JWT 版）

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity   // 开启方法级权限注解
public class SecurityConfig {

    @Autowired
    private JwtAuthenticationFilter jwtAuthenticationFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))  // 无状态
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/login", "/register").permitAll()        // 放行登录注册
                .anyRequest().authenticated()
            );

        // JWT 过滤器加在用户名密码过滤器之前
        http.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public AuthenticationManager authenticationManager(AuthenticationConfiguration config)
            throws Exception {
        return config.getAuthenticationManager();
    }
}
```

## 7. 方法级权限控制

配置类加 `@EnableMethodSecurity` 后，在方法上直接声明权限：

```java
@PreAuthorize("hasAuthority('sys:user:list')")
@GetMapping("/user/list")
public Result list() { ... }

@PreAuthorize("hasRole('ADMIN')")
@DeleteMapping("/user/{id}")
public Result delete(@PathVariable Long id) { ... }
```

### 常用权限注解

| 注解 | 说明 |
|------|------|
| `@PreAuthorize` | 方法执行**前**校验，最常用 |
| `@PostAuthorize` | 方法执行**后**校验返回值 |
| `@Secured` | 基于角色校验（需单独开启） |

### `hasAuthority` vs `hasRole`

```
hasAuthority("sys:user:list")  → 精确匹配权限字符串
hasRole("ADMIN")               → 匹配 ROLE_ADMIN（框架自动加前缀）
```

## 常见问题

| 问题 | 原因 / 解决 |
|------|------------|
| 403 Forbidden | 已认证但无权限，检查 `getAuthorities()` 返回值是否正确 |
| 401 Unauthorized | 未认证，检查 JWT 过滤器是否正确写入 SecurityContextHolder |
| 循环依赖 | `SecurityConfig` 注入 `UserDetailsService` 时，改用 `@Lazy` 或通过 `AuthenticationConfiguration` 获取 Manager |
| 密码匹配失败 | 确认数据库存的是 BCrypt 加密后的密码，而非明文 |
