## Java 参数校验（Bean Validation）

### 常用注解

| 注解                 | 说明                                  |
| ------------------ | ----------------------------------- |
| `@NotNull`         | 值不能为 `null`，但允许空字符串                 |
| `@NotEmpty`        | 值不能为 `null` 且长度 > 0（适用于字符串、集合）      |
| `@NotBlank`        | 值不能为 `null`，且去除空白字符后长度 > 0（仅适用于字符串） |
| `@Size(min, max)`  | 字符串长度 / 集合大小必须在 `[min, max]` 范围内    |
| `@Min(value)`      | 数值最小值                               |
| `@Max(value)`      | 数值最大值                               |
| `@Pattern(regexp)` | 字符串必须匹配正则表达式                        |
| `@Email`           | 字符串必须是合法 Email 格式                   |

### 示例

```java
@NotBlank(message = "登录账号不能为空")
@Size(min = 2, max = 16, message = "账号长度必须在2-16之间")
private String username;

@NotBlank(message = "密码不能为空")
private String password;
```

> 注意：同一字段重复标注相同注解（如两个 `@NotBlank`）是多余的，只保留一个即可。

### 三者区别

```
@NotNull  → null 不通过，"" 通过，" " 通过
@NotEmpty → null 不通过，"" 不通过，" " 通过
@NotBlank → null 不通过，"" 不通过，" " 不通过  ✅ 最严格，推荐用于字符串字段
```

### 使用步骤

1. **引入依赖**（Spring Boot 已内置）

2. **在 DTO/实体类字段上加注解**（如上示例）

3. **在 Controller 方法参数上加 `@Valid` 或 `@Validated`**

   ```java
   @PostMapping("/login")
   public Result login(@RequestBody @Valid LoginDTO dto) {
       // ...
   }
   ```

4. **全局异常处理**（捕获校验失败异常）

   ```java
   @RestControllerAdvice
   public class GlobalExceptionHandler {

       @ExceptionHandler(MethodArgumentNotValidException.class)
       public Result handleValidException(MethodArgumentNotValidException e) {
           String msg = e.getBindingResult().getFieldErrors().stream()
               .map(FieldError::getDefaultMessage)
               .collect(Collectors.joining("; "));
           return Result.fail(msg);
       }
   }
   ```

### `@Valid` vs `@Validated`

| | `@Valid` | `@Validated` |
|---|---|---|
| 来源 | JSR-303 标准 | Spring 扩展 |
| 分组校验 | 不支持 | 支持 |
| 嵌套校验 | 支持（加在字段上） | 支持 |
| 常用场景 | 一般校验 | 需要分组时使用 |
