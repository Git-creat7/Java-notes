# 回顾
## Malloc
`malloc` 的全称是 **Memory Allocation**（内存分配）。它是 C 语言中从**堆（Heap）**区域手动申请内存的核心函数
```C
	void* malloc(size_t size);
```
- **参数 `size`**：想要申请的内存字节数。通常配合 `sizeof` 使用，以确保跨平台兼容性。
    
- **返回值**：
    
    - 成功：返回分配到的内存**首地址**（`void*` 类型，可以强转为任何类型）。
    - 失败：返回 `NULL`

|**特性**|**栈 (Stack)**|**堆 (Heap) / malloc**|
|---|---|---|
|**大小**|较小（通常几 MB）|很大（取决于系统内存）|
|**控制权**|系统自动分配/释放|程序员手动分配/释放|
|**灵活性**|固定大小，编译时确定|**动态确定**，运行中可变|

# 线性表
由 n(n>0)个数据特性相同的元素构成的有限序列，称为线性表。
- 线性表是 n 个数据元素的有限序列，其中 n个数据是相同数据类型的
- 线性表中元素的个数 n(n≥0)定义为线性表的长度，当 n=0 时称之为空表。
- 对于非空的线性表或线性结构，其特点是:
	- **存在唯一的一个被称作“第一个”的数据元素**
	- **存在唯一的一个被称作“最后一个”的数据元素**
	- **除第一个元素外，结构中的每个数据元素均只有一个前驱**
	- **除最后一个元素外，结构中的每个数据元素均只有一个后继**

## 顺序表
用一组连续的内存单元依次存储线性表的各个元素，也就是说，逻辑上相邻的元素，实际的物理存储空间也是连续的。
### 初始化顺序表
```C
#include <stdio.h>
#define MAXSIZE 100
typedef int ElemType;
typedef struct {
    ElemType data[MAXSIZE];
    int length;
}SeqList;
void initList(SeqList *L){
    L->length = 0;
}
int main(void){
    SeqList L; initList(&L);
    printf("初始化成功，目前长度占用：%d\n", L.length);
    printf("目前占用内存大小：%d\n", sizeof(L.data));
    return 0;
}
```
### 在尾部添加元素
```C
int appendElem(SeqList *L,ElemType e){ //尾插法
    if(L->length == MAXSIZE) {
        printf("顺序表已满");
        return 0;
    }  
    L -> data[L -> length] = e;
    L -> length ++;
    return 1;
}
```
### 插入元素
`时间复杂度 O(1) ~ O(n)`
```C
int insertElem(SeqList *L, int pos, ElemType e){ //pos：位置
    if(L->length == MAXSIZE){
        printf("顺序表已满");
        return 0;
    }
    if(pos < 1 || pos > L->length + 1){
        printf("位置不合法");
        return 0;
    }
    for(int i = L->length; i >= pos; i--){
        L->data[i] = L->data[i-1];
    }
    L->data[pos-1] = e;
    L->length++;
    return 1;
}
```
### 删除元素
```C
int deleteElem(SeqList *L,int pos,ElemType *e){
    if(L->length == 0 || pos < 1 || pos > L->length) 
    return 0;
    //e = 地址 *e = 值   
    *e = L -> data[pos-1];
    for(int i = pos; i < L->length;i++)
        L->data[i-1] = L -> data[i];
    L->length --;
    return 1;
}
```

### 查找元素
```C
int findElem(SeqList *L,ElemType e){
    for(int i = 0; i < L->length; i++){
        if(L -> data[i] == e){
            return i+1;
        }
    }
    return 0;
}
```
### 动态分配内存地址初始化
```C
typedef struct {
	ElemType *data;
	int length;
}SeqList;
SeqList* initList(){
    SeqList *L = (SeqList*)malloc(sizeof(SeqList));
    L -> data = (ElemType*)malloc(sizeof(ElemType)*MAXSIZE);
    L -> length = 0;
    return L;
}
```
- **第一次 `malloc(sizeof(SeqList))`**：在堆上开辟了一块空间，用来存放 `data` 指针、`length` 等成员。
	    
- **第二次 `malloc(sizeof(ElemType)*MAXSIZE)`**：在堆上开辟了一块连续的数组空间，并将它的首地址赋给 `p->data`
- 这样就可以传入list了：
- ```C
	int main(void){
	SeqList* list = initList();
	appendElem(list,32);
	listElem(list);
	return 0;
	}
   ```


# 链表
- 链表中的结点是独立的对象，在内存中是不连续的，每个节点包含数据值和下一个结点的地址
- 链表==查询慢==，无论查询哪个数据都从头开始找。
- 链表的==增删快==（对比数组）
## 节点
在链表中，数据不再是连续存放的，而是分散在内存的各个角落。每个数据块被称为一个**节点**。
一个节点由两部分组成：
1. **数据域**：存你想存的东西（如 `int`）。
    
2. **指针域**：存**下一个节点**的地址（那个“钩锁”）
```C
Node* initList(){
    Node* head = (Node*)malloc(sizeof(Node));
    head -> data = 0;
    head -> next = NULL;
    return head;
}
```
别名：增加可读性
```C
typedef struct Node {
    ElemType data;
    struct Node *next;
} Node, *LinkedList; // LinkedList 等价于 Node*
```
如果这样写了，`main` 函数里就可以写成： `LinkedList list = initList();` 
这样从字面上看，`list` 就代表了“整个链表”，而不仅仅是一个节点指针了。
**但在底层，它依然是一个 `Node*`**
## 头插法
为了保证链表不断裂，这两行代码的**顺序**是绝对不能错的：

1. `p -> next = L -> next;`
    
    **逻辑**：让新节点 `p` 先牵住原先排在第一位的那个人。如果后面原本没节点（`NULL`），那么 `p->next` 就指向 `NULL`。
    
2. `L -> next = p;`
    
    **逻辑**：让头结点 `L` 改手牵住新节点 `p`
```C
int insertHead(Node* L,ElemType e){
	Node* p = (Node*)malloc(sizeof(Node));
	p -> data = e;
	p -> next = L -> next;
	L -> next = p;
}
```
假设依次插入 `10`, `20`, `30`：
1. 插入 10：`Head -> 10 -> NULL`
    
2. 插入 20：`Head -> 20 -> 10 -> NULL`
    
3. 插入 30：`Head -> 30 -> 20 -> 10 -> NULL`


## 尾插法
要实现尾插，最关键的是找到当前链表的**最后一个节点**。

1. 创建一个指针 `r`（取名为 rear，意为尾部），初始指向头结点。
    
2. 通过 `while` 循环让 `r` 走到尽头（即 `r->next == NULL` 的地方）。
    
3. 把新节点挂在 `r` 的后面
```C
int insertTail(LinkList L,ElemType e){
    //创建新节点
    LinkList h = (LinkList)malloc(sizeof(Node));
    if(h == NULL) return 0;
    h -> data = e;
    h - > next = NULL;
    //找到尾节点
    Node* r = L;
    while(r -> next != NULL){
        r = r -> next;
    }
    //将新节点插入到尾节点后面
    r -> next = h;
    return 0;
}
```
假设链表现在是 `Head -> 10 -> NULL`，要插入 `20`：

- **找尾巴**：`r` 发现 `r->next` 指向 10，于是 `r` 变成 10。10 的 `next` 是 `NULL`，循环停止，`r` 现在就是 10。
    
- **挂载**：执行 `r->next = p`，即 10 的 `next` 指向了 20。
    
- **结果**：`Head -> 10 -> 20 -> NULL`

# 数组

- ==添加效率低==：添加位置后的每个数据后移，再添加元素

## 二叉树的遍历方式
### 前序遍历

- **顺序：根节点 -> 左子树 -> 右子树**
    
- **描述：** 只要遇到节点，先访问它，然后递归地去处理它的左子树，最后处理右子树。
    
- **特点：** 根节点总是第一个被访问的。
    

### 中序遍历 

- **顺序：左子树 -> 根节点 -> 右子树**
    
- **修正：** 并不是简单地“从最左边开始”，而是**先完整地遍历左子树**，再访问当前节点，最后遍历右子树。
    
- **特点：** 对于**二叉搜索树 (BST)**，中序遍历的结果是一个严格递增的序列。
    

### 3. 后序遍历

- **顺序：左子树 -> 右子树 -> 根节点**
    
- **修正：** 它的逻辑是**先完成左右子树的访问**，最后才回到当前节点。
    
- **特点：** 根节点总是最后一个被访问的。常用于文件目录的大小计算或删除整棵树。
    

### 4. 层序遍历 

- **顺序：按层从上到下，每层从左到右**
    
- **描述：** 借助**队列 (Queue)** 实现。先将根节点入队，然后依次弹出节点，并将该节点的左右子节点入队。

|**遍历方式**|**访问顺序简记**|**核心逻辑**|
|---|---|---|
|**前序**|**根** - 左 - 右|遇到就拿，先根后后|
|**中序**|左 - **根** - 右|左边到底，中间插一杠|
|**后序**|左 - 右 - **根**|左右搞定，最后回根|
|**层序**|按层遍历|广度优先 (BFS)|

## 二叉查找树
- 每一个节点上最多有两个子节点
- 任意节点左子树上的值都小于当前节点
- 任意节点右子树上的值都大于当前节点

### 添加节点
- 小的存左边，大的存右边，一样的不存
### 弊端
- 左子树全部为空，从形式上看，更像一个==单链表==
- 插入速度没有影响
- 查询速度明显降低

### 解决弊端：平衡二叉树
平衡二叉树也叫 **平衡二叉搜索树（Self-balancing binary search tree）**，又被称为 AVL 树，可以保证 **查询效率较高**。它是解决 **二叉排序** 可能出现的查询问题。

 - 二叉查找树
 - 规则：==**任意**节点左右子树高度差不超过1==

#### 保持平衡的旋转机制
- 左旋
- 右旋
- 当添加一个节点时，该树不再是一颗平衡二叉树 -> 旋转
#### 左旋
- 确定支点：从添加的节点开始，不断的往父节点找不平衡的节点

- 步骤：
	- 以不平衡的点作为支点
	- 把支点左旋降级，变成左子节点
	- 晋升 原来的右子节点

#### 需要旋转的四种情况
##### 左左LL
- 当==根结点左子树的左子树==有节点插入，导致二叉树不平衡
- 一次右旋
##### 左右LR
- 当==根结点左子树的右子树==有节点插入，导致二叉树不平衡
- 先局部左旋 -> 左左LL
- 再整体右旋
##### 右右RR
- 当==根结点右子树的右子树==有节点插入，导致二叉树不平衡
- 一次左旋
##### 右左RL
- 当==根结点右子树的左子树==有节点插入，导致二叉树不平衡
- 先局部右旋 -> 右右RR
- 再整体左旋

## 红黑树
- 平衡二叉树的弊端：在添加节点的时候，旋转次数太多，造成添加节点的时间浪费
- 红黑树刚开始被称为**平衡二叉B树**
- 是一种特殊的二叉查找树，红黑树的每一个节点上都有存储位表示节点的颜色
- <span style="color: #dc3545;">每一个节点可以是红或者黑；红黑树不是高度平衡的，他的平衡是通过“红黑规则“进行实现的</span>
- 是一个二叉查找树
- ==但是不是高度平衡的==
- 条件：特有的<span style="color: #dc3545;">红黑规则</span>
### 红黑规则
1. 每一个节点是红色或黑色
2. 根节点必须是黑色的
3. 如果一个节点没有子节点或者父节点，则该节点相应的指针属性值为Nil，这些Nil被视为叶节点，每个叶节点 (Nil) 是黑色的
4. 如果某一个节点是红色的，那么它的子节点必须是黑色（不能出现两个红色节点相连的情况）
5. 对每一个节点，从该接待你到其所有后代叶节点的简单路径上，均包含相同数目的黑色节点

#### 添加节点
- 添加的节点默认是红色的（红色效率高）
- 查找、插入、删除的时间复杂度最坏为O(log n)
