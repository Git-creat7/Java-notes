## 第 1 题：Toast 显示用户输入 ⭐ | 10分 | Test-2 Q1

### 题目还原
```
EditText(id:et1) + Button(id:but1, onClick:show)
点击按钮 → 取输入内容 → Toast显示
```

### 答案（两行代码）
```java
public void show(View view) {
    String input = ((EditText) findViewById(R.id.et1)).getText().toString().trim();
    Toast.makeText(this, input, Toast.LENGTH_SHORT).show();
}
```

### 手把手拆解
```
第1行：((EditText) findViewById(R.id.et1))  // 找到编辑框 → 强转类型
                          .getText()        // 取文本
                          .toString()       // 转字符串
                          .trim();          // 去首尾空格

第2行：Toast.makeText(this,         // 上下文
                       input,        // 要显示的内容
                       Toast.LENGTH_SHORT) // 显示时长
          .show();   // 别忘了调用show()！！
```

### 记忆技巧
- **口诀**：找控件 → 取文本 → makeText → show
- Toast 三参数：**this, 内容, LENGTH_SHORT**
- `.show()` 必须写，不写 Toast 不出来！

### 容易错
| 错误写法 | 问题 |
|----------|------|
| `findViewById(R.id.et1)` 不加 `(EditText)` | 类型不匹配，没法调 getText() |
| `Toast.makeText(this, input, Toast.LENGTH_SHORT)` 不加 `.show()` | Toast 创建了但不显示 |
| `findViewById(R.id.but1)` | 找错 id，题目要求的是 et1 |
| `.toString()` 写成 `.toStirng()` | 拼写错误 |

### 阅卷老师看什么
- (EditText) 强转有没有 → 1分
- .getText().toString().trim() 链式调用是否完整 → 2分
- Toast.makeText 三个参数是否正确 → 2分
- .show() 是否写了 → 1分

## 第 2 题：初始化 List<Student> ⭐ | 10分 | Test-2 Q2

### 题目还原
```java
private List<Student> stus;
private void initDatas() {
    // 往stus里添加10条记录
}
// Student 有 sno(学号) 和 sname(姓名) 两个属性
```

### 答案
```java
private void initDatas() {
    stus = new ArrayList<>();
    for (int i = 1; i <= 10; i++) {
        stus.add(new Student("S" + i, "学生" + i));
    }
}
```

### 手把手拆解
```
第1步：stus = new ArrayList<>();     ← 必须初始化，否则空指针
第2步：for (int i = 1; i <= 10; i++) ← 标准循环，从1到10
第3步：new Student("S"+i, "学生"+i)  ← 学号="S1"~"S10"，姓名="学生1"~"学生10"
第4步：stus.add(...)                 ← 添加到列表
```

### 记忆技巧
- **三件套**：new ArrayList → for 1到10 → new Student + add
- Student 构造参数顺序：`Student(sno, sname)` → 学号在前，姓名在后

### 容易错
| 错误 | 后果 |
|------|------|
| 忘记 `stus = new ArrayList<>()` | NullPointerException |
| `for (int i = 0; i < 10; i++)` | 生成的是 0~9，不是 1~10 |
| `new Student("学生"+i, "S"+i)` | 参数顺序反了 |

### 阅卷老师看什么
- 是否初始化了 new ArrayList → 3分
- 循环次数是否正确(10条) → 3分
- Student 构造参数是否合理 → 4分

## 第 3 题：AlertDialog 退出确认 ⭐⭐ | 10分 | Test-4 Q1

### 题目还原
```
按下返回键 → 弹对话框 → 标题"提示" → 内容"确定要退出吗？"
→ 确定按钮(退出) + 取消按钮 → 点确定执行 finish()
```

### 答案
```java
@Override
public void onBackPressed() {
    new AlertDialog.Builder(this)
            .setTitle("提示")                    // 空1
            .setMessage("确定要退出吗？")          // 空2
            .setPositiveButton("确定", new DialogInterface.OnClickListener() {
                @Override
                public void onClick(DialogInterface dialog, int which) {
                    finish();                    // 空4
                }
            })                                   // 空3：setPositiveButton
            .setNegativeButton("取消", null)
            .create()     // 或者不写create直接show
            .show();                             // 空5
}
```

### 五个空的答案
```
空1：setTitle
空2：setMessage
空3：setPositiveButton
空4：finish();
空5：show()
```

### 手把手拆解
```
Builder 链式调用顺序（固定！不能换）：
  new AlertDialog.Builder(this)
  → .setTitle("...")        // 标题
  → .setMessage("...")      // 内容
  → .setPositiveButton()    // 确定按钮（右边）
  → .setNegativeButton()    // 取消按钮（左边）
  → .create()               // 创建对话框对象
  → .show()                 // 显示出来

确定按钮的回调里写：finish();  ← 退出当前Activity
取消按钮直接传 null 就行       ← 点取消自动关闭
```

### 记忆技巧
- **口诀**：标题 → 内容 → 确定 → 取消 → 显示
- 方法名规律：set + 英文名 → Title / Message / PositiveButton / NegativeButton
- **"Positive" = "积极的/肯定的" = 确定按钮**
- **"Negative" = "消极的/否定的" = 取消按钮**
- 确定退出的逻辑 = `finish()`
- 如果用 `create()` 就要再调 `.show()`；如果直接用 `.show()` 就不用 create

### 容易错
| 错误 | 说明 |
|------|------|
| `setTitle` 写成 `setTitile` | 拼写 |
| `setPositiveButton` 写成 `setPositiveBtn` | 方法名必须完整 |
| 确定回调里写 `System.exit(0)` | Android 退出不这样，用 finish() |
| 忘了 `.show()` | 对话框创建了但不显示 |

### 阅卷老师看什么
- 五个方法名拼写是否正确 → 每空 2 分
- 确定按钮里是否调了 finish() → 关键逻辑分
- 顺序：Title→Message→Positive→Negative→show 不能乱

## 第 4 题：Activity 跳转传参 ⭐⭐ | 10分 | Test-4 Q2

### 题目还原
```
ActivityA: 编辑框(et1) → 读内容 → 跳转到ActivityB，把内容传过去
ActivityB: 接收数据 → 显示在 TextView(tv1)
```

### 答案
```java
// ===== ActivityA：发送数据 =====
EditText et1 = findViewById(R.id.et1);
String content = et1.getText().toString().trim();
Intent intent = new Intent(ActivityA.this, ActivityB.class);
intent.putExtra("input_data", content);
startActivity(intent);

// ===== ActivityB：接收数据 =====
TextView tv1 = findViewById(R.id.tv1);
Intent intent = getIntent();
if (intent != null) {
    String receivedData = intent.getStringExtra("input_data");
    tv1.setText(receivedData);
}
```

### 手把手拆解
```
【发送端 ActivityA】
第1步：找到 EditText → findViewById
第2步：取出文本 → getText().toString().trim()
第3步：创建 Intent → new Intent(当前Activity.this, 目标Activity.class)
第4步：放入数据 → intent.putExtra("key", value)
第5步：跳转 → startActivity(intent)

【接收端 ActivityB】
第1步：找到 TextView → findViewById (如果需要显示)
第2步：获取 Intent → getIntent()
第3步：判断非空 → if (intent != null)
第4步：取出数据 → intent.getStringExtra("key")  ← key要和发送端一致！
第5步：显示 → tv1.setText(数据)
```

### 记忆技巧
- **发送口诀**：findId → 取文本 → new Intent → putExtra → startActivity
- **接收口诀**：findId → getIntent → 判空 → getStringExtra → setText
- putExtra：存入键值对（存入）
- getStringExtra：按键取值（取出）
- **key 必须一模一样**，建议题目给什么就用什么

### putExtra / getXXXExtra 对照表
| 存入方法 | 取出方法 | 数据类型 |
|----------|----------|----------|
| putExtra("k", String) | getStringExtra("k") | String |
| putExtra("k", int) | getIntExtra("k", 默认值) | int |
| putExtra("k", boolean) | getBooleanExtra("k", 默认值) | boolean |

### 容易错
| 错误 | 后果 |
|------|------|
| `new Intent(ActivityB.this, ActivityA.class)` | 反了！应该是 A→B |
| `putExtra` 的 key 是 `"input_data"`，但 `getStringExtra` 的 key 是 `"data"` | 取不到，返回 null |
| ActivityB 里用 `new Intent()` 而不是 `getIntent()` | 拿到空 Intent |
| ActivityB 里忘记判空 | 如果 intent 为 null 会崩溃 |

### 阅卷老师看什么
- Intent 构造的两个参数是否正确 (类名 + .class) → 2分
- putExtra 的 key 是否合理 → 1分
- ActivityB 是否用 getIntent() 而不是 new Intent() → 2分
- getStringExtra 的 key 是否与发送一致 → 1分
- 是否判空 → 1分
- 最终 setText → 1分

## 第 5 题：Fragment 动态加载 ⭐⭐ | 5分 | Test-7 Q2

### 题目还原
```
ActivityA里有一个FragmentContainerView(id: fcv)
请写出动态加载MyFragment的代码
```

### 答案
```java
getSupportFragmentManager()
    .beginTransaction()
    .replace(R.id.fcv, new MyFragment())
    .commit();
```

### 手把手拆解
```
只有三行，固定写法：
getSupportFragmentManager()        // 获取 Fragment 管理器
  .beginTransaction()              // 开启事务
  .replace(R.id.fcv,               // 替换容器（id）
           new MyFragment())       // 替换成新的 Fragment 实例
  .commit();                       // 提交事务
```

### 记忆技巧
- **口诀**：管理器 → 开事务 → replace(容器id, new Fragment()) → commit
- `.replace()` 三个要素：容器 id + Fragment 实例

| 错误 | 说明 |
|------|------|
| `getFragmentManager()` 而不是 `getSupportFragmentManager()` | 老版本用前者，新版本用后者，看题目上下文 |
| `.replace(R.id.fcv, ...)` id 写错 | 题目给的是 fcv |
| 忘记 `.commit()` | Fragment 不会显示 |

### 阅卷老师看什么

## 第 6 题：ListView + BaseAdapter ⭐⭐ | 20分 | Test-1

### 题目还原
```
ListView(id:lv1) 显示 Student 列表
自定义 Adapter 继承 BaseAdapter
加载布局：stu_item，包含两个TextView(id:tv1, tv2)
```

### 完整答案
```java
class StuAdapter extends BaseAdapter {
    private List<Student> mData;
    private Context mContext;

    public StuAdapter(Context context, List<Student> data) {
        this.mContext = context;
        this.mData = data;
    }

    @Override
    public int getCount() {
        return mData.size();                    // 空1（2分）
    }

    @Override
    public Object getItem(int position) {
        return mData.get(position);             // 空2（2分）
    }

    @Override
    public long getItemId(int position) {
        return position;                        // 空3（2分）
    }

    @Override
    public View getView(int position, View convertView, ViewGroup parent) {
        ViewHolder holder;
        if (convertView == null) {
            convertView = LayoutInflater.from(mContext)
                .inflate(R.layout.stu_item, parent, false);  // 空4 前半（4分）
            holder = new ViewHolder();
            holder.tv1 = convertView.findViewById(R.id.tv1); // 空4 中段（3分）
            holder.tv2 = convertView.findViewById(R.id.tv2);
            convertView.setTag(holder);                      // 空4 中段2（1分）
        } else {
            holder = (ViewHolder) convertView.getTag();      // 空4 后半（2分）
        }
        Student s = mData.get(position);
        holder.tv1.setText(s.getName());
        holder.tv2.setText(s.getStatus());
        return convertView;
    }

    static class ViewHolder {                   // 需要声明ViewHolder类
        TextView tv1, tv2;
    }
}

// 设置适配器（4分）
lv1.setAdapter(new StuAdapter(this, stus));      // 空5
```

### 手把手拆解 — getView 的"如果null否则"模式

这是整道题最关键的部分（10分大空），把流程记住：

```
convertView 是什么？
  → 屏幕上滑出视野的旧 item，系统回收给你复用
  → 如果是 null：说明没有旧的可复用，需要新建
  → 如果不是 null：直接拿来用，只更新数据

【是 null → 新建】
1. inflate 加载布局 → 得到一个 View
2. new ViewHolder → 创建容器
3. holder.tv1 = view.findViewById(R.id.tv1) → 找子控件
4. holder.tv2 = view.findViewById(R.id.tv2)
5. view.setTag(holder) → 把 ViewHolder 存到 View 上，下次复用

【不是 null → 复用】
6. holder = (ViewHolder) convertView.getTag() → 取出之前存好的 ViewHolder

【共同部分 — 不管是不是 null 都要做】
7. 从 mData 取当前位置的 Student
8. holder.tv1.setText(...) → 更新数据
9. holder.tv2.setText(...)
10. return convertView → 返回这个 View，显示到屏幕上
```

### 五个必背值
```
空1: mData.size()           ← getCount 返回集合大小
空2: mData.get(position)    ← getItem 返回当前位置元素
空3: position               ← getItemId 返回位置作为id
空4: LayoutInflater.from(mContext).inflate(R.layout.stu_item, parent, false)
     setTag(holder) / getTag()
空5: lv1.setAdapter(new StuAdapter(this, stus))
```

### ViewHolder 三句不可少
```java
convertView.setTag(holder);               // 存
holder = (ViewHolder) convertView.getTag(); // 取
static class ViewHolder { TextView tv1, tv2; } // 定义
```

### 记忆技巧
- **getCount** → 问"有多少条" → 答 `size()`
- **getItem** → 问"第i条是什么" → 答 `get(i)`
- **getItemId** → 问"第i条的id" → 答 `i`
- **getView** → 问"这个位置怎么显示" → inflate + ViewHolder 模式
- **inflate 三参数**：布局id → parent → **false**（一定是 false！）

### 容易错 — 排行榜
| 排名 | 错误 | 后果 |
|------|------|------|
| 🥇 | inflate 第三个参数写 true | 运行时报错 |
| 🥈 | 忘记 setTag / getTag | ViewHolder 优化失效，列表卡顿 |
| 🥉 | ViewHolder static 忘了 | 语法警告 |
| 4 | findViewById 的 id 写错(tv1→tv2) | 数据显示错位 |
| 5 | getCount 返回 0 | 列表一片空白 |
| 6 | 忘记 setAdapter | 列表不显示 |

### 阅卷老师看什么
- getCount/getItem/getItemId 三个方法是否正确(2分×3=6分)
- getView 是否有 null 判断 + inflate(4分)
- 是否有 ViewHolder 优化 setTag/getTag(2分)
- 是否正确绑定数据(2分)
- 是否设置了 setAdapter(4分)

## 第 7 题：RecyclerView + Adapter ⭐⭐⭐ | 20分 | Test-3

### 题目还原
```
RecyclerView(id:rv1) 显示数据
加载布局：stu_item，包含两个TextView(id:tv1, tv2)
```

### 完整答案
```java
class RvAdapter extends RecyclerView.Adapter<RvAdapter.MyHolder> {
    private List<Stu> dataList;

    public RvAdapter(List<Stu> dataList) {
        this.dataList = dataList;
    }

    @NonNull
    @Override
    public MyHolder onCreateViewHolder(@NonNull ViewGroup parent, int viewType) {
        // 空1（4分）
        View view = LayoutInflater.from(parent.getContext())
                .inflate(R.layout.stu_item, parent, false);
        return new MyHolder(view);
    }

    @Override
    public void onBindViewHolder(@NonNull MyHolder holder, int position) {
        // 空2（4分）
        Stu stu = dataList.get(position);
        holder.tv1.setText(stu.getName());
        holder.tv2.setText(stu.getInfo());
    }

    @Override
    public int getItemCount() {
        // 空3（2分）
        return dataList == null ? 0 : dataList.size();
    }

    class MyHolder extends RecyclerView.ViewHolder {
        // 空4（2分）→ 声明变量
        TextView tv1, tv2;

        public MyHolder(@NonNull View itemView) {
            super(itemView);
            // 空5（2分）→ findViewById
            tv1 = itemView.findViewById(R.id.tv1);
            tv2 = itemView.findViewById(R.id.tv2);
        }
    }
}

// 设置 RecyclerView（6分）
RecyclerView rv1 = findViewById(R.id.rv1);                        // 空6-1（1分）
rv1.setLayoutManager(new LinearLayoutManager(this));              // 空6-2（3分）
RvAdapter adapter = new RvAdapter(stuList);                       // 空6-3（1分）
rv1.setAdapter(adapter);                                          // 空6-4（1分）
```

### ListView vs RecyclerView 对照记忆

| 步骤 | ListView | RecyclerView |
|------|----------|-------------|
| 基类 | extends BaseAdapter | extends RecyclerView.Adapter<XXX.MyHolder> |
| 创建 View | 在 getView 里 inflate | 在 onCreateViewHolder 里 inflate |
| 绑定数据 | 在 getView 里 setText | 在 onBindViewHolder 里 setText |
| 返回数量 | getCount() | getItemCount() |
| ViewHolder | 手动写 setTag/getTag | 强制继承 RecyclerView.ViewHolder |
| 设置列表 | 直接 setAdapter | setLayoutManager + setAdapter |

### 手把手拆解 — 三个核心方法

```
【onCreateViewHolder】— 只在需要新建 item 时调用
参数: parent(父容器), viewType(类型)
要做:
  1. inflate 加载布局 → 得到一个 View
  2. 把这个 View 传给 ViewHolder 构造
  3. return new MyHolder(view)

【onBindViewHolder】— 每个 item 显示时都调用
参数: holder(ViewHolder), position(位置)
要做:
  1. 从 dataList 取第 position 个数据
  2. 往 holder 的控件 setText

【getItemCount】— 返回总数
要做: return dataList.size()  （判空更稳妥）
```

### ViewHolder 内部 — 跟 ListView 不一样
```java
class MyHolder extends RecyclerView.ViewHolder {
    TextView tv1, tv2;                          // 1.先声明

    public MyHolder(@NonNull View itemView) {
        super(itemView);                        // 2.必须调父类构造
        tv1 = itemView.findViewById(R.id.tv1); // 3.用 itemView 找控件
        tv2 = itemView.findViewById(R.id.tv2); //   不是 convertView！
    }
}
```

### RecyclerView 设置三步走 — 最容易漏！
```java
// 这三行缺一不可！缺任何一行列表都不显示
rv1.setLayoutManager(new LinearLayoutManager(this));  // ← 最容易忘！
rv1.setAdapter(new RvAdapter(dataList));
// 注意：先 setLayoutManager 再 setAdapter，顺序无所谓但两个都要有
```

### 记忆技巧
- **三个方法**：onCreateViewHolder(建) → onBindViewHolder(绑) → getItemCount(数)
- **ViewHolder 三部曲**：声明 → super(itemView) → findViewById(itemView)
- **设置两部曲**：setLayoutManager → setAdapter
- **数据判空**：`dataList == null ? 0 : dataList.size()` 比直接 size() 更安全

### 容易错 — 排行榜
| 排名 | 错误 | 后果 |
|------|------|------|
| 🥇 | 忘记 `setLayoutManager` | RecyclerView 一片空白，最常见的坑 |
| 🥈 | ViewHolder 构造中忘记 `super(itemView)` | 编译错误 |
| 🥉 | `onBindViewHolder` 写成 `onBindViewHoler` | 拼写错误，不算重写 |
| 4 | ViewHolder 中用 `findViewById` 而不是 `itemView.findViewById` | 找不到控件 |
| 5 | `onCreateViewHolder` 忘记 `return` | 返回 null |
| 6 | inflate 用 `mContext` 而不是 `parent.getContext()` | 两者都可以，但 parent.getContext() 更推荐 |

### 阅卷老师看什么
- onCreateViewHolder 是否 inflate + return ViewHolder(4分)
- onBindViewHolder 是否正确绑定数据(4分)
- getItemCount 是否正确(2分)
- ViewHolder 声明变量(2分)
- ViewHolder 构造中 findViewById(2分)
- 设置 RecyclerView 时是否包含 LayoutManager(3分) + setAdapter(3分)

## 第 8 题：SQLite 数据库操作 ⭐⭐⭐ | 20分 | Test-9

### 题目还原
```
① 创建 user.db 数据库，版本1（2分）
② 建表 user(name, password)（4分）
③ 升级：添加 state 列，不丢原有数据（4分）
④ 登录验证：用 et_name 和 et_pwd 查 user 表（10分）
```

### 完整答案

```java
// ===== SQLiteOpenHelper 类 =====
public class MyHelper extends SQLiteOpenHelper {

    public MyHelper(@Nullable Context context) {
        // 空①：数据库名 + 版本号（2分）
        super(context, "user.db", null, 1);
    }

    @Override
    public void onCreate(SQLiteDatabase db) {
        // 空②：建表（4分）
        String sql = "CREATE TABLE user("
                   + "_id INTEGER PRIMARY KEY AUTOINCREMENT, "
                   + "name TEXT, "
                   + "password TEXT)";
        db.execSQL(sql);
    }

    @Override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
```

## 第 9 题：JSON 解析 ⭐⭐⭐ | 10分 | Test-6 Q2

### 题目还原
```json
{"data":[{desc, id, title, url},{...}],"errorCode":0,"errorMsg":""}
```
完成方法 `List<Article> parseArticle(String strJson)`

实体类字段：`desc(String)`, `id(int)`, `title(String)`, `url(String)`

### 答案
```java
List<Article> parseArticle(String strJson) {
    List<Article> list = new ArrayList<>();

    try {
        // 1. 拿到根对象
        JSONObject root = new JSONObject(strJson);

        // 2. 判断是否成功
        int errorCode = root.getInt("errorCode");
        if (errorCode == 0) {

            // 3. 拿到 data 数组
            JSONArray dataArray = root.getJSONArray("data");

            // 4. 遍历数组
            for (int i = 0; i < dataArray.length(); i++) {
                JSONObject obj = dataArray.getJSONObject(i);

                // 5. 逐个字段取出，赋值
                Article article = new Article();
                article.setDesc(obj.getString("desc"));
                article.setId(obj.getInt("id"));
                article.setTitle(obj.getString("title"));
                article.setUrl(obj.getString("url"));

                list.add(article);
            }
        }
    } catch (JSONException e) {
        e.printStackTrace();
    }

    return list;
}
```

### 手把手拆解
```
JSON 解析五步走：

① new JSONObject(字符串) → 把字符串变成 JSON 对象
② getInt/getString → 取基本字段
③ getJSONArray("data") → 取数组字段
④ for 循环 + getJSONObject(i) → 遍历数组里的每个对象
⑤ getString/getInt → 取对象里的字段

嵌套结构对应关系：
{                          ← JSONObject
  "errorCode": 0,          ← getInt("errorCode")
  "data": [                ← getJSONArray("data")
    {                      ← getJSONObject(0) → JSONObject
      "desc": "xxx",       ← getString("desc")
      "id": 29,            ← getInt("id")
      "title": "xxx",      ← getString("title")
      "url": "xxx"         ← getString("url")
    }
  ]
}
```

### 记忆技巧
- `{` → JSONObject
- `[` → JSONArray
- `"key": "字符串值"` → getString("key")
- `"key": 数字` → getInt("key")
- 必须 try-catch `JSONException`
- 解析前判断 `errorCode == 0`

### 容易错
| 错误 | 后果 |
|------|------|
| `id` 用 getString 而不是 getInt | 类型不匹配 |
| 忘记 try-catch | 编译错误 |
| 遍历前忘记判断 errorCode | 失败时也解析了 |
| `getJSONArray` 写成 `getJSONArary` | 拼写错误 |

### 阅卷老师看什么
- JSONObject 构造是否正确(2分)
- errorCode 判断(2分)
- getJSONArray 使用正确(2分)

## 第 10 题：startActivityForResult ⭐⭐⭐⭐ | 20分 | Test-5

### 题目还原（标注已补全答案）
```
ActivityA: TextView(tv1) + Button(btn, onClick:getPhone)
  点击 → 打开 ActivityB 获取电话 → 显示在 tv1

ActivityB: EditText(et1) + Button(btn, onClick:retPhone)
  点击 → 获取输入内容 → 返回给 ActivityA
```

### 完整答案

```java
// ==================== ActivityA ====================
public class ActivityA extends AppCompatActivity {

    private TextView tv1;                                  // 空1：声明变量

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_a);
        tv1 = findViewById(R.id.tv1);                     // 空2：初始化
    }

    public void getPhone(View view) {                     // 空3：点击方法
        Intent intent = new Intent(ActivityA.this, ActivityB.class);
        startActivityForResult(intent, 100);               // 100 是请求码
    }

    @Override
    protected void onActivityResult(int requestCode, int resultCode, @Nullable Intent data) {
        super.onActivityResult(requestCode, resultCode, data);
        if (requestCode == 100 && resultCode == RESULT_OK) {  // 空4：判断
            String phone = data.getStringExtra("phone");      // 取出数据
            tv1.setText(phone);                                // 显示
        }
    }
}

// ==================== ActivityB ====================
public class ActivityB extends AppCompatActivity {

    private EditText et1;                                 // 空5：声明变量

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_b);
        et1 = findViewById(R.id.et1);                     // 空6：初始化
    }

    public void retPhone(View view) {                     // 空7：点击方法
        String phone = et1.getText().toString().trim();
        Intent intent = new Intent();
        intent.putExtra("phone", phone);
        setResult(RESULT_OK, intent);                     // 核心：设置返回结果
        finish();                                          // 关闭自己
    }
}
```

### 手把手拆解 — 三个关键机制

#### 机制1：A 发起带返回的跳转
```java
startActivityForResult(intent, 100);
//                        ↑      ↑
//                     Intent   请求码（自己定，一般从100开始）
```

#### 机制2：B 设置返回数据
```java
Intent intent = new Intent();          // 注意：不要传目标Activity！
intent.putExtra("phone", phone);       // 放入数据
setResult(RESULT_OK, intent);          // 设置结果码 + 数据
finish();                              // 关闭B，回到A
```

**关键点**：
- `new Intent()` 不传任何参数！不需要指定目标
- `setResult` 两个参数：结果码 + Intent
- `finish()` 必须调用，否则不会返回

#### 机制3：A 接收返回数据
```java
@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    // 三个参数：
    // requestCode → A发出时的请求码，用来识别是哪次跳转
    // resultCode  → B设置的结果码(RESULT_OK或RESULT_CANCELED)
    // data        → B传回的Intent，里面装着数据

    super.onActivityResult(requestCode, resultCode, data);  // 必须调super

    if (requestCode == 100 && resultCode == RESULT_OK) {    // 两个条件都判断
        String phone = data.getStringExtra("phone");         // 取出数据
        tv1.setText(phone);
    }
}
```

### 完整流程图
```
A: startActivityForResult(intent, 100)
   ↓
B: 用户输入电话 → 点按钮
   ↓
B: intent.putExtra("phone", phone)
   setResult(RESULT_OK, intent)
   finish()
   ↓
B 销毁，A 重新出现
   ↓
A: onActivityResult(100, RESULT_OK, intent含phone)
   取出 phone，显示到 tv1
```

### 记忆技巧
- **A端公式**：`startActivityForResult(Intent, 请求码)`
- **B端公式**：`new Intent() → putExtra → setResult(RESULT_OK, intent) → finish()`
- **A端接收公式**：`onActivityResult → 判requestCode + 判resultCode → getStringExtra`
- **两个码**：requestCode(自己定) + resultCode(用系统的RESULT_OK)

### 容易错 — 致命错误榜
| 排名 | 错误 | 后果 |
|------|------|------|
| 🥇 | B 中 `new Intent(ActivityB.this, ActivityA.class)` | 不是 startActivity！不需要指定目标 |
| 🥈 | B 忘记 `setResult(RESULT_OK, intent)` | A 收不到数据 |
| 🥉 | B 忘记 `finish()` | 页面不关闭，回不到 A |
| 4 | A 用 `startActivity` 而不是 `startActivityForResult` | onActivityResult 永远不会被调用 |
| 5 | onActivityResult 中只判断 requestCode 不判断 resultCode | 取消操作也会触发 |
| 6 | putExtra 的 key 和 getStringExtra 的 key 不一致 | 取到 null |
| 7 | onActivityResult 中忘记 `super.onActivityResult(...)` | 系统警告 |

## 第 11 题：HttpURLConnection POST ⭐⭐⭐⭐ | 10分 | Test-6 Q1

### 题目还原
```java
// 完成 POST 请求方法，传 username 和 password，返回服务器响应字符串
String httpPost(String url, String name, String pwd) { }

// 调用在子线程中
new Thread() {
    public void run() {
        String str = httpPost(url, name, pwd);
        // ...
    }
}.start();
```

### 答案
```java
String httpPost(String url, String name, String pwd) {
    try {
        // 1. 创建 URL 对象
        URL u = new URL(url);

        // 2. 打开连接
        HttpURLConnection conn = (HttpURLConnection) u.openConnection();

        // 3. 设置 POST
        conn.setRequestMethod("POST");
        conn.setDoOutput(true);   // 允许写出数据
        conn.setDoInput(true);    // 允许读入数据

        // 4. 拼接参数
        String params = "username=" + URLEncoder.encode(name, "UTF-8")
                      + "&password=" + URLEncoder.encode(pwd, "UTF-8");

        // 5. 写出参数
        OutputStream os = conn.getOutputStream();
        os.write(params.getBytes());
        os.close();

        // 6. 读取响应
        InputStream is = conn.getInputStream();
        BufferedReader reader = new BufferedReader(new InputStreamReader(is));
        StringBuilder sb = new StringBuilder();
        String line;
        while ((line = reader.readLine()) != null) {
            sb.append(line);
        }
        reader.close();

        // 7. 返回
        return sb.toString();

    } catch (Exception e) {
        e.printStackTrace();
    }
    return null;
}
```

### 手把手拆解 — POST 请求七步法

```
口诀：开→设→拼→写→读→返→关

第1步：new URL(url) → 创建 URL 对象
第2步：url.openConnection() → 强转为 HttpURLConnection
第3步：conn.setRequestMethod("POST")
       conn.setDoOutput(true)   → 要写数据出去
第4步：拼接参数字符串 "username=xxx&password=xxx"
       用 URLEncoder.encode() 编码中文
第5步：conn.getOutputStream() → 写出参数 → close
第6步：conn.getInputStream() → BufferedReader逐行读取 → StringBuilder拼接
第7步：return sb.toString()
```

### 记忆技巧
- **POST 和 GET 的区别**：
  - GET：参数拼在 URL 后面 `?key=value`，不用写 OutputStream
  - POST：参数通过 OutputStream 发送，`setDoOutput(true)`
- **URLEncoder.encode**：处理特殊字符（中文、空格等）
- **BufferedReader 读法**：`while((line=reader.readLine())!=null)` 是固定写法
- 题目告诉你在子线程调用 → 不需要写 new Thread()，只是让你知道可以写耗时操作

### 容易错 — 步骤遗漏榜
| 排名 | 错误 | 后果 |
|------|------|------|
| 🥇 | 忘记 `conn.setDoOutput(true)` | 无法写参数 |
| 🥈 | URLEncoder.encode 忘记写 | 中文参数乱码或报错 |
| 🥉 | 忘记 close() OutputStream | 内存泄漏 |
| 4 | `setRequestMethod` 写成 `setRequestMothed` | 拼写 |
| 5 | InputStream 和 OutputStream 搞混 | 读写反了 |

### 阅卷老师看什么
- URL 创建 + openConnection(2分)
- POST 设置 + setDoOutput(2分)
- 参数拼接 + URLEncoder(2分)
- 写出参数(2分)
- 读取响应 + return(2分)

### 重要补充：Handler 线程通信（往年真题必考）

POST 请求在子线程执行，但更新 UI 必须在主线程。**Handler 用于从子线程向主线程发消息**。

#### 完整示例（网络请求 + Handler）

```java
public class ActivityB extends AppCompatActivity {
    private TextView textView;

    // 1. 主线程定义 Handler
    private Handler handler = new Handler() {
        @Override
        public void handleMessage(@NonNull Message msg) {
            // 这里在主线程执行，可以更新 UI
            String result = (String) msg.obj;
            textView.setText(result);
        }
    };

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_b);
        textView = findViewById(R.id.textView);
        
        // 2. 子线程执行网络请求
        new Thread(() -> {
            String data = httpPost(url, name, pwd);  // 网络请求
            
            // 3. 发消息给主线程
            Message msg = Message.obtain();
            msg.obj = data;
            handler.sendMessage(msg);
        }).start();
    }
}
```

#### Handler 三步记忆

```
第1步：主线程定义 Handler { handleMessage(msg) { 更新UI } }
第2步：子线程 Message.obtain() + msg.obj = 数据
第3步：handler.sendMessage(msg)
```

#### 为什么需要 Handler？

| | 主线程 | 子线程 |
|---|--------|--------|
| 可以做 | 更新 UI、处理事件 | 网络请求、文件读写 |
| 不能做 | 耗时操作（会卡顿） | 更新 UI（会崩溃） |

Android 规定只能在主线程更新 UI，Handler 是跨线程通信的桥梁。

## 第 13 题：Service + Binder + Broadcast ⭐⭐⭐⭐⭐ | 20分 | Test-8

### 题目还原（全卷最难，已补全答案）
```
MyService：包含 MyBinder + play()方法（发广播带参数zzuli）
ActivityA：
  1. 启动服务
  2. 调用 play()
  3. 定义 BroadcastReceiver 接收广播，显示到 tv1
```

### 完整答案

```java
// ==================== MyService ====================
public class MyService extends Service {

    public MyService() { }

    public class MyBinder extends Binder {
        // 空1：返回 Service 实例，让 Activity 能调用服务的方法
        public MyService getService() {
            return MyService.this;
        }
    }

    @Override
    public IBinder onBind(Intent intent) {
        return new MyBinder();
    }

    @Override
    public int onStartCommand(Intent intent, int flags, int startId) {
        // 空2：服务启动时的逻辑（如非必要，保持默认即可）
        return super.onStartCommand(intent, flags, startId);
    }

    public void play() {
        // 空3（3分）：发送广播，携带字符串 zzuli
        Intent intent = new Intent("com.example.PLAY_ACTION");
        intent.putExtra("key", "zzuli");
        sendBroadcast(intent);
    }
}

// ==================== ActivityA ====================
public class ActivityA extends AppCompatActivity {

    private TextView tv1;
    private MyService myService;          // 保存 Service 引用
    private MyReceiver receiver;
    private ServiceConnection connection;
    private boolean isBound = false;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        tv1 = findViewById(R.id.tv1);

        // 空4（3分）：启动服务
        Intent serviceIntent = new Intent(this, MyService.class);
        startService(serviceIntent);

        // 绑定服务（为了能调用 play 方法）
        connection = new ServiceConnection() {
            @Override
            public void onServiceConnected(ComponentName name, IBinder service) {
                MyService.MyBinder binder = (MyService.MyBinder) service;
                myService = binder.getService();  // 拿到 Service 实例
                isBound = true;
            }

            @Override
            public void onServiceDisconnected(ComponentName name) {
                isBound = false;
            }
        };
        bindService(serviceIntent, connection, Context.BIND_AUTO_CREATE);

        // 空6 准备工作：注册广播接收者
        receiver = new MyReceiver();
        IntentFilter filter = new IntentFilter("com.example.PLAY_ACTION");
        registerReceiver(receiver, filter);

        // 空5（10分）：调用 play 方法
        // 假设某处触发（如按钮点击），实际调用：
        if (myService != null) {
            myService.play();
        }
        // 注意：由于 bindService 是异步的，myService 可能还没赋值
        // 考试中简单写法可直接：
        // myService.play();
    }

    @Override
    protected void onDestroy() {
        // 空6 准备：注销广播接收者
        unregisterReceiver(receiver);
        // 解绑服务
        if (isBound) {
            unbindService(connection);
        }
        super.onDestroy();
    }

    // 空6（4分）：广播接收者
    class MyReceiver extends BroadcastReceiver {
        @Override
        public void onReceive(Context context, Intent intent) {
            String data = intent.getStringExtra("key");
            tv1.setText(data);
        }
    }
}
```

### 手把手拆解 — 三组件联动流程

```
【整体流程】
Activity 启动 Service → 绑定 Service → 拿到 Binder
→ 通过 Binder 调用 play()
→ play() 发广播
→ BroadcastReceiver 收到广播 → 更新 UI

【组件关系图】
ActivityA ──startService──→ MyService (启动)
ActivityA ──bindService───→ MyService (绑定，拿到 Binder)
ActivityA ──binder.play()─→ MyService.play() 执行
MyService ──sendBroadcast─→ 系统广播中心
系统广播中心 ──分发──→ MyReceiver.onReceive()
MyReceiver ──setText──→ tv1 更新
```

#### 块1：Binder 暴露 Service

```java
    public MyService getService() {
        return MyService.this;
    }

```

#### 块2：发送广播

```java
sendBroadcast(intent);
```

#### 块3：启动服务
```java

```

#### 块4：绑定服务

### 记忆技巧 — 故事法

```
有一家餐厅（Activity）：
1. 厨房（Service）开着做菜 → startService
2. 服务员（Binder）是连接厨房和餐厅的人 → onBind 返回 Binder
3. 点菜（play）通过服务员传到厨房 → binder.getService().play()
4. 菜做好了，厨房摇铃（发广播）→ sendBroadcast
5. 餐厅的耳朵（BroadcastReceiver）听到铃声去取菜 → onReceive 更新UI
6. 餐厅关门要关厨房、解雇服务员、捂上耳朵 → stopService + unbindService + unregisterReceiver
```

### 容易错 — 全卷失分王者
| 排名 | 错误 | 后果 |
|------|------|------|
| 🥇 | Binder 没写 getService() 方法 | Activity 拿不到 Service，无法调用 play |
| 🥈 | 发广播的 action 和接收的 IntentFilter 字符串不一致 | 永远收不到广播 |
| 🥉 | 忘记 registerReceiver | 接收不到广播 |
| 4 | 忘记 unregisterReceiver | 内存泄漏 |
| 5 | 忘记在 AndroidManifest.xml 注册 Service | Service 无法启动 |
| 6 | 用 `new Intent(this, MyService.class)` 发广播 | 广播 Intent 不需要指定目标类 |
| 7 | startService 和 bindService Intent 用错 | 启动失败 |

## 第 14 题：ListView 点击事件 ⭐⭐⭐ | 往年真题必考

### 题目类型

点击列表项跳转到详情页，并携带数据。

### 标准答案

```java
ListView listView = findViewById(R.id.lv);
listView.setAdapter(adapter);

// 设置点击监听
listView.setOnItemClickListener(new AdapterView.OnItemClickListener() {
    @Override
    public void onItemClick(AdapterView<?> parent, View view, int position, long id) {
        // parent: ListView 本身
        // view: 被点击的 item 的 View
        // position: 点击的位置（从0开始）← 最重要！
        // id: 点击的 item 的 id
        
        // 1. 用 position 取出点击的数据
        Student student = students.get(position);
        
        // 2. 跳转并传参
        Intent intent = new Intent(MainActivity.this, DetailActivity.class);
        intent.putExtra("name", student.getName());
        intent.putExtra("age", student.getAge());
        startActivity(intent);
    }
});
```

### 往年真题示例（2022-2023）

```java
// 点击书籍列表，跳转到 BookActivity，携带 link
listView.setOnItemClickListener(new AdapterView.OnItemClickListener() {
    @Override
    public void onItemClick(AdapterView<?> parent, View view, int position, long id) {
        Book book = books.get(position);  // 用 position 取数据
        Intent intent = new Intent(ActivityC.this, BookActivity.class);
        intent.putExtra("link", book.getLink());
        startActivity(intent);
    }
});
```

### 手把手拆解

```
第1步：listView.setOnItemClickListener(监听器)
第2步：onItemClick 方法四个参数记住 position
第3步：数据.get(position) 取出点击的对象
第4步：new Intent 跳转
第5步：putExtra 传参
第6步：startActivity
```

### 记忆技巧

- **四个参数只记 position**（其他三个基本不用）
- **position 是下标**，从 0 开始，用它从数据源取数据
- 取数据 + 跳转 + 传参 是固定三件套

### 容易错

| 错误 | 后果 |
|------|------|
| 用 `id` 而不是 `position` 取数据 | id 和下标不一定一致，可能取错 |
| `students.get(id)` | 应该用 `position` |
| 忘记 `startActivity(intent)` | Intent 创建了但没跳转 |

### 阅卷老师看什么

- setOnItemClickListener 调用（2分）
- onItemClick 方法签名（1分）
- 用 position 取数据（2分）
- new Intent + 传参（3分）
- startActivity（2分）
