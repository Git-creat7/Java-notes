## 逐题详细分析

### Test-1：ListView + BaseAdapter（20分）⭐⭐

**题型**：5个填空（2+2+2+10+4）

**空位清单**：
1. `getCount()` 返回什么 → `return mData.size();`
2. `getItem()` 返回什么 → `return mData.get(position);`
3. `getItemId()` 返回什么 → `return position;`
4. `getView()` 整段代码 → inflate + ViewHolder + 数据绑定
5. ListView 设置适配器 → `lv1.setAdapter(...)`

**难度分析**：
- 四个方法签名、ViewHolder 模式全是固定模板，零变化
- 第4空虽然是10分大空，但结构固定：null判断 → inflate → findViewById → setTag/getTag → 绑定数据 → return convertView
- **这是全卷最容易拿满分的题目**，因为只有一种标准写法
- 唯一可变项：布局名 `stu_item`、id `tv1` `tv2`、数据类名 `Student` — 但题目都会给出

**结论**：送分题，背熟模板即可。预计丢分 ≤2 分。

### Test-2 Q1：Toast 显示输入内容（10分）⭐

**题型**：完整代码编写

```java
public void show(View view) {
    String input = ((EditText) findViewById(R.id.et1)).getText().toString().trim();
    Toast.makeText(this, input, Toast.LENGTH_SHORT).show();
}
```

**难度分析**：
- 就两行代码
- EditText 取值 + Toast 显示，最基础操作
- findViewById 强转类型容易忘写 `(EditText)`

**结论**：全卷最简单。唯一陷阱是忘加强转。

---

### Test-2 Q2：初始化 List 数据（10分）⭐

**题型**：完成 `initDatas()` 方法，往 stus 添加10条 Student

```java
stus = new ArrayList<>();
for (int i = 1; i <= 10; i++) {
    stus.add(new Student("S" + i, "学生" + i));
}
```

**难度分析**：
- 基础 Java 循环 + List.add
- 注意题目答案中 `String.format("S"+i)` 写法有些多余，直接 `"S"+i` 就够了
- 唯一易错：忘记 `stus = new ArrayList<>();`

**结论**：纯 Java 基础，与 Android 无关。送分。

---

### Test-3：RecyclerView + Adapter（20分）⭐⭐⭐

**题型**：6个填空（4+4+2+2+2+6）

**空位清单**：
1. `onCreateViewHolder` → inflate + return new MyHolder(view)
2. `onBindViewHolder` → 数据绑定
3. `getItemCount` → return size
4. ViewHolder 成员变量声明 → `TextView tv1, tv2;`
5. ViewHolder 构造中 findViewById → `tv1 = itemView.findViewById(R.id.tv1);`
6. 设置 LayoutManager + Adapter → `setLayoutManager(new LinearLayoutManager(this))` + `setAdapter(...)`

**难度分析**：
- 也是模板题，但比 ListView 多一步：必须设置 LayoutManager
- 三个方法各有固定写法，ViewHolder 内部结构固定
- **最容易忘**：第6空的 `LinearLayoutManager` — 很多考生省略导致列表不显示
- 需要区分 `onCreateViewHolder`（创建）和 `onBindViewHolder`（绑定）的职责

**结论**：模板化程度高但步骤比 ListView 多，属于"背了就有分，不背全没分"。

### Test-4 Q1：AlertDialog 退出确认（10分）⭐⭐

**题型**：5个填空（每空2分）

**空位清单**：
1. `setTitle("提示")`
2. `setMessage("确定要退出吗？")`
3. `setPositiveButton("确定", listener)`
4. 确定按钮回调 → `finish();`
5. `.show()`

**难度分析**：
- 纯链式调用模板，顺序固定
- 五个空对应 Builder 模式的五个方法
- 唯一需要理解的是第4空 → 确定后要退出，所以是 `finish()`

**结论**：送分题。Dialog 的 Builder 链式调用只会这一种写法。

---

### Test-4 Q2：Activity 跳转传参（10分）⭐⭐

**题型**：两个独立代码段（5+5）

**A端发送**：
```java
Intent intent = new Intent(ActivityA.this, ActivityB.class);
intent.putExtra("input_data", content);
startActivity(intent);
```

**B端接收**：
```java
Intent intent = getIntent();
String received = intent.getStringExtra("input_data");
tv1.setText(received);
```

**难度分析**：
- 两个方向各三行，套路固定
- 唯一可变：key 字符串名 — 但题目会用同一个名字
- 不需要处理 null（题目默认有数据）

**结论**：简单。注意 key 前后一致就行。

### Test-5：startActivityForResult（20分）⭐⭐⭐⭐

**题型**：两个 Activity 共 7 个编号位置填代码

**代码框架**：
```
ActivityA: ①声明常量 → ②findViewById → ③startActivityForResult → ④onActivityResult中取数据
ActivityB: ⑤findViewById → ⑥获取输入+putExtra+setResult+finish → ⑦retPhone方法
```

**难度分析**：
- **这是第一个真正的"难"题**，原因：
  1. 涉及两个 Activity 协作
  2. 需要理解回调机制（onActivityResult 不是你自己调用的）
  3. 需要区分 `startActivity` 和 `startActivityForResult` 的使用场景
  4. requestCode/RESULT_OK 这些常量需要记住
  5. 题目标注"编号处不一定需要填写代码"增加不确定性
- 考生常犯错误：忘记 `setResult(RESULT_OK, intent)`，导致 B 返回了但 A 收不到
- 另一个坑：把 `startActivityForResult` 写成 `startActivity`

**结论**：中等偏难。两组件交互 + 回调理解，比单向传参高一个档次。这是区分 60 分段和 80 分段的关键题。

### Test-6 Q1：HttpURLConnection POST（10分）⭐⭐⭐⭐

**题型**：完成 `httpPost(String url, String name, String pwd)` 方法

**需要写的代码**：
1. 创建 URL 对象
2. 打开连接、设置 POST
3. 拼接表单参数
4. 写出参数到 OutputStream
5. 读取 InputStream 返回字符串

**难度分析**：
- 步骤多（至少 5-6 个关键步骤）
- 需要 import 多个类（URL, HttpURLConnection, URLEncoder, BufferedReader...）
- 需要 try-catch 包裹
- `URLEncoder.encode()` 容易忘
- 但题目给了调用格式（在子线程中调用），说明不需要写线程代码
- I/O 流操作容易写错顺序

**结论**：中偏难。步骤固定但多，考试紧张时容易漏步骤。好在只要写过一次就记住了。

### Test-6 Q2：JSON 解析（10分）⭐⭐⭐

**题型**：完成 `parseArticle(String strJson)` → 返回 `List<Article>`

**需要解析的 JSON**：
```json
{"data":[{desc, id, title, url},{...}],"errorCode":0,"errorMsg":""}
```

**答案结构**：
1. `new JSONObject(strJson)` 拿到根对象
2. `getInt("errorCode")` 判断是否为 0
3. `getJSONArray("data")` 拿到数组
4. 循环、`getJSONObject(i)`、逐个 `getString/getInt` 赋值给 Article
5. try-catch JSONException

**难度分析**：
- JSONObject/JSONArray API 固定
- 关键是知道解析顺序：Object → Array → Object → 字段
- errorCode 判断逻辑要写对
- 类似题目实验5做过，有参考

**结论**：中等。解析流程套路化，比 POST 网络请求简单。

### Test-7 Q1：ContentObserver（15分）⭐⭐⭐⭐

**题型**：3个填空（5+2+8） — **没有明确答案**

**空位**：
1. onCreate 中注册 ContentObserver → `registerContentObserver`
2. onDestroy 中注销 → `unregisterContentObserver`
3. onChange 中查询数据并更新 UI

**难度分析**：
- ContentObserver 全卷只出现过这一次，是低频考点
- 需要知道 Uri 解析和 Cursor 查询
- 第 3 空 8 分需要写查询 + 更新 UI 代码，步骤较多
- `ContentResolver` 的 `registerContentObserver` 和 `query` API 需要记住

**结论**：中偏难。低频 + 查询步骤多。但由于模板固定，背下来也能拿分。

### Test-7 Q2：Fragment 动态加载（5分）⭐⭐

**题型**：写出动态加载 MyFragment 到 fcv 的代码

**答案**：

getSupportFragmentManager()

### Test-8：Service + Binder + Broadcast（20分）⭐⭐⭐⭐⭐

**题型**：6个填空分布在 Service 和 Activity — **全卷最难**

**Service 端**：
1. MyBinder 类中返回 Service 实例
2. onStartCommand 方法体
3. play 方法中发送广播（3分）
4. startService 启动（3分）
5. 通过 Binder 调用 play（10分）
6. BroadcastReceiver 接收广播更新 UI（4分）

**难度分析**：
- **三道坎**：
  1. Binder 通信：需要 ServiceConnection 回调中获取 Binder，很多考生搞不清流程
  2. 广播收发：Service 发 → Activity 收，两端的 action 字符串必须一致
  3. 三种组件联动：Service + BroadcastReceiver + Activity 串在一起
- 第 5 空 10 分，是整张卷子单空最高分 — 需要写 bindService + ServiceConnection + 调用 play
- 需要理解 `onBind` 返回值如何传到 Activity
- 容易遗漏：忘记在 AndroidManifest.xml 中注册 Service（虽然填空可能不考这个）

**结论**：全卷最难。多组件协作 + Binder 通信机制，这是区分 85+ 和 90+ 的题目。

### Test-9：SQLite 完整操作（20分）⭐⭐⭐

**题型**：三个小问 + 一段登录验证代码

1. ① 创建 user.db 数据库（2分）
2. ② 建表 user(name, password)（4分）
3. ③ onUpgrade 添加 state 列（4分）
4. ④ 登录验证查询代码（10分）

**难度分析**：
- 第 ① 问最简单：`super(context, "user.db", null, 1);`
- 第 ② 问需要写完整 CREATE TABLE SQL，容易漏 `_id INTEGER PRIMARY KEY AUTOINCREMENT`
- 第 ③ 问 ALTER TABLE ADD COLUMN 是固定写法，但很多人不知道该用 ALTER
- 第 ④ 问 10 分需要完整流程：helper → getWritableDatabase → query → cursor.moveToFirst → 判断

**结论**：中等。SQL 是固定的，比网络请求容易。但 onUpgrade 的 ALTER TABLE 容易和 onCreate 的 CREATE TABLE 搞混。

## 整体难度分布

```
送分题（⭐-⭐⭐）:    Test-2(20分) + Test-1(20分) + Test-4(20分) = 60分
中等题（⭐⭐⭐）:     Test-3(20分) + Test-9(20分) + Test-6 Q2(10分) = 50分  
难题（⭐⭐⭐⭐-⭐⭐⭐⭐⭐）: Test-5(20分) + Test-6 Q1(10分) + Test-7 Q1(15分) + Test-8(20分) = 65分
```

> 总题量约 175 分（测试题总分），如果考试取 100 分，预计送分/中等/难题的比例约为 **35:30:35**

## 关键结论

| 结论 | 说明 |
|------|------|
| **最容易拿满分的** | Test-1(ListView)、Test-2(Toast)、Test-4(Dialog+Intent) — 纯模板 |
| **最可能丢大分的** | Test-8(Service) — 10分大空写不出来直接没 |
| **区分度最高的** | Test-5(startActivityForResult) — 中等生和优等生的分水岭 |
| **低频但分值高** | Test-8(Service广播) — 就考这一次但 20 分 |
| **最容易马虎丢分** | Test-4 Q2 Intent key 不一致、Test-3 忘写 LayoutManager |
| **需要 Java 功底** | Test-6(网络+I/O)、Test-9(SQL) — 不纯是 Android |

**策略建议**：先确保拿满 60 分送分题（模板背熟），再攻坚中等题，最后有时间再啃 Service/Broadcast 难题。
