## 1. 网页的组成部分及常见 HTML 标签

## 2. CSS 引入方式与选择器优先级

<details>
<summary>参考答案</summary>

**引入方式：**
- **行内样式** — `style` 属性直接写在元素上，仅对当前元素生效
- **内部样式** — `<head>` 中的 `<style>` 标签，仅对当前页面生效
- **外部样式** — 独立 `.css` 文件通过 `<link>` 引入，复用性最强

**选择器写法：**
- 元素选择器：`div`
- ID 选择器：`#example`
- 类选择器：`.example`

**优先级：** ID 选择器 > 类选择器 > 元素选择器

</details>

## 11. HTTP 协议及特点

<details>
<summary>参考答案</summary>

**HTTP 协议：** 超文本传输协议，规定浏览器与服务器之间数据传输的规则。

**特点：**
- 基于 TCP，面向连接，安全
- 基于请求-响应模型，一次请求一次响应
- 无状态，每次请求独立，不携带前次请求数据

</details>

## 12. HTTP 请求与响应的数据格式

<details>
<summary>参考答案</summary>

**请求格式：**
- 请求行：请求方式 + 请求路径 + 协议版本
- 请求头：key:value 键值对
- （空行）
- 请求体：POST/PUT 请求发送的数据

**响应格式：**
- 响应行：协议版本 + 状态码 + 状态消息
- 响应头：key:value 键值对
- （空行）
- 响应体：服务器返回的数据

</details>

## 18. char 与 varchar 的区别

<details>
<summary>参考答案</summary>

| 对比项 | char | varchar |
|--------|------|---------|
| 存储方式 | 定长，始终占用固定空间 | 变长，按实际长度占用空间 |
| 性能 | 操作更快 | 相对慢一点 |

**选择：** 长度固定（如手机号）用 char，长度不固定（如姓名）用 varchar。

</details>

## 28. 事务

<details>
<summary>参考答案</summary>

**定义：** 一系列操作要么全部成功，要么全部失败。

**SQL 控制：**
```sql
start transaction; / begin;
commit;
rollback;
```

**四大特性（ACID）：**
- 原子性 — 不可分割，全部成功或全部失败
- 一致性 — 事务完成后数据保持一致
- 隔离性 — 事务间互不影响
- 持久性 — 提交/回滚后数据改变是永久的

**Spring 中：** 在业务方法上加 `@Transactional` 注解。

</details>

## 34. Cron 表达式

<details>
<summary>参考答案</summary>

**格式：** 秒 分 时 日 月 周 (年)

**示例：** `5 10 1 * * ?` → 每天凌晨 1:10:05 执行

**特殊符号：**
- `*` — 任意
- `?` — 放弃指定（日和周互斥时使用）

- `-` — 范围
- `,` — 枚举

</details>

## 39. AOP 面向切面编程

<details>
<summary>参考答案</summary>

将与业务无关但对多个对象产生影响的公共行为（日志、事务、权限）抽取为公共模块，降低耦合。

**项目实践（操作日志）：**
1. 自定义注解 `@Log`，加在需要记录日志的方法上
2. 定义切面类，用环绕通知 + `@annotation` 切点表达式
3. 获取类信息、方法信息、请求参数、操作人、时间等，保存到数据库

</details>

## 44. SpringBoot 自动配置原理

<details>
<summary>参考答案</summary>

1. `@SpringBootApplication` → `@EnableAutoConfiguration` → `@Import` 导入 ImportSelector 实现类
2. 读取 `META-INF/spring/xxx.imports` 文件中配置的自动配置类全类名
3. 配置类中通过 `@Bean` 定义 Bean，结合 `@Conditional` 系列注解（如 `@ConditionalOnClass`）按条件决定是否注册

</details>
