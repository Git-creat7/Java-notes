## 下一步

### 第一步：@Valid 参数校验（1-2 小时）

给实体类加校验注解：
- `Emp`: `@NotBlank(name)`, `@Pattern(phone, email)`, `@Size` 等
- `Dept`: `@NotBlank(deptName)` 等
- `Student`: `@NotBlank(name)`, 关联班级 ID 校验
- `Clazz`: `@NotBlank(name)`, 日期范围校验

**新增和修改接口都要加校验**。如果新增和修改的校验规则不同，用校验分组：
```java
// 定义分组
public interface ValidationGroups {
    interface Insert {}
    interface Update {}
}

// 实体类中指定分组
@NotBlank(message = "姓名不能为空", groups = {Insert.class, Update.class})
private String name;

// Controller 中使用
@PostMapping
public Result insertEmp(@Validated(Insert.class) @RequestBody Emp emp) { ... }

@PutMapping
public Result updateEmp(@Validated(Update.class) @RequestBody Emp emp) { ... }
```

GlobalExceptionHandler 补充处理：
```java
@ExceptionHandler(MethodArgumentNotValidException.class)
public Result handleValidation(MethodArgumentNotValidException e) {
    String msg = e.getBindingResult().getFieldErrors().stream()
        .map(FieldError::getDefaultMessage)
        .collect(Collectors.joining(", "));
    return Result.error(msg);
}
```

**验证方式**：用 curl 或 Postman 发一个缺少必填字段的请求（如 `POST /emps` 不传 name），确认返回类似 "姓名不能为空" 的错误信息，HTTP 200 但 Result.code 为错误码。

### 第二步：AOP 操作日志切面（1 天）

对初学者来说，这一步包含多个子任务，建议分配 1 天时间，不要赶进度。

**子步骤 1：修改 TokenFilter，存储用户信息**

当前 TokenFilter 只验证 token 但不存储用户信息。需要在验证通过后把 Claims 存到 request attribute：

```java
// TokenFilter.doFilter 中，验证通过后加这行：
Claims claims = JwtUtils.parseJwt(token);
httpRequest.setAttribute("claims", claims);
```

边界情况：token 过期时 `parseJwt` 会抛异常，当前代码已经 catch 了，不会影响。

**子步骤 2：自定义注解 + 切面**

1. 自定义注解 `@OperationLog(module="员工", operation="新增")`
2. 写切面类，用 `@Around` 拦截 **Service 层方法**（不是 Controller 层，因为 Service 层能拿到业务对象）。用 `@Around` 而不是 `@Before` + `@AfterReturning`，因为需要记录耗时，必须控制方法执行前后
3. 获取操作人时，先做 null 检查防止 NPE：
```java
ServletRequestAttributes attrs = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
if (attrs == null) {
    // 非 HTTP 请求（如定时任务），跳过记录
    return joinPoint.proceed();
}
Claims claims = (Claims) attrs.getRequest().getAttribute("claims");
```
4. 记录方法名、参数、返回值、耗时、IP 地址（`request.getRemoteAddr()`）
5. 存入数据库，使用 `TEXT` 类型存 JSON 字符串（比 MySQL JSON 列更通用，学习阶段够用）
6. params 和 result 超过 2000 字符时截断，避免单行过大

**子步骤 3：建表 + 替换旧逻辑**

1. 创建 operation_log 表
2. 替换掉 `EmpServiceImpl` 里硬编码的 `EmpLog` 记录。旧写法注释掉，确认新切面工作后再删除。`EmpLog` 表保留不删（里面有历史数据），新数据写入 operation_log
3. 给所有写操作（新增/修改/删除）的 Service 方法加上注解

**操作日志表结构**：
```sql
CREATE TABLE operation_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    operator VARCHAR(50) COMMENT '操作人',
    module VARCHAR(50) COMMENT '模块名',
    operation VARCHAR(50) COMMENT '操作类型',
    method VARCHAR(200) COMMENT '方法名',
    params TEXT COMMENT '请求参数(JSON)',
    result TEXT COMMENT '返回结果(JSON)',
    ip VARCHAR(50) COMMENT '操作人IP',
    duration_ms BIGINT COMMENT '耗时(毫秒)',
    created_at DATETIME COMMENT '操作时间'
);
```

**注意 Spring AOP 的限制**：Spring AOP 只能拦截通过代理对象的外部调用。如果同一个 Service 类里的方法 A 调用了方法 B，方法 B 上的 `@OperationLog` 不会被拦截。如果需要记录内部调用，把方法抽到另一个 Bean 里。

**验证方式**：调用 `POST /emps` 新增一个员工，检查 operation_log 表：operator 非空，module='员工'，operation='新增'，params 包含请求体内容，duration_ms > 0，ip 非空。

- 所有 Controller 的 `@RequestBody` 都有 `@Valid` 校验，新增和修改都覆盖
