# 💡 一句话核心

> 哪边是 Outer，哪边就是“亲儿子”，数据必须全出来；对面的匹配不上就填 NULL。


**Q:  既然 Left 和 Right 只是换个位置，为什么大家基本都只用 LEFT JOIN？**

**A:** 
1. **符合人类阅读习惯：** 我们习惯从左往右看，把“主表”放在左边逻辑更顺。 

2. **代码一致性：** 统一使用 `LEFT JOIN` 可以让复杂的 SQL（连 5、6 张表时）结构更清晰，不容易把自己绕晕。

3. **互换性：** `A LEFT JOIN B` 等价于 `B RIGHT JOIN A`。所以，掌握一个就够了，生产环境推荐 **全员 LEFT JOIN**。

# 踩坑记录

- **现象：** 做了 LEFT JOIN 结果行数比原表多了？
    
- **原因：** 右表（从表）中存在多条记录对应左表的一条记录（一对多关系）。
    
- **解药：** 检查关联条件（ON 后面的字段）是否唯一，或者是否需要使用 `DISTINCT` 或 `GROUP BY`。
# 犯的错误

题目要求：**查询拥有员工的部门**

- **INNER JOIN** = 只查两边都匹配的数据（最适合本题）
- **RIGHT JOIN** 会把没有部门的员工也带进来，产生 NULL 部门
- 题目要**有员工的部门** → 必须用 **INNER JOIN** 或加 **IS NOT NULL**
